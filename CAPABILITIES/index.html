    Crucible Testbed — Capabilities & Limitations¶

    Purpose: A clear, honest assessment of what the Crucible testbed can and cannot do. This document helps researchers, reviewers, and users understand the boundaries of the system before investing time in it.

    Quick Summary¶

    Dimension Status Detail
    Benchmark attacks ✅ 5 of 6 pass All at L2 fork-state fidelity
    L4 bridge demos ✅ 2 of 2 pass nomad_bridge_l4, demo_agent (with --no-fidelity)
    Version ✅ v0.4.0 Tagged release — clean git history with professional commits
    Unit tests ✅ 782 passed, 0 failed No regressions, all passing
    Documentation ✅ 28 docs, comprehensive User guide, CLI, troubleshooting, fidelity, RPC, architecture, etc.
    Determinism checks ✅ 4 of 5 verified wormhole, nomad, horizon, bnb_token_hub all deterministic; poly_network has archive-RPC variability across 3 chains
    Cross-provider verification ❌ Skipped No second RPC provider available (free tiers only); see code-level analysis below
    Publication bundles ✅ 4 exported All passing experiments exported with integrity PASS; wormhole/nomad have 24 files each, horizon/bnb have 18 files each
    Artifact protection ✅ Permanent by default clean --all never touches logs/ or datasets/; --logs respects tiers; --datasets requires --confirm; publish adds .protected marker
    CI/CD ❌ Removed Local-only project — tests run on-device, no CI pipeline configured

    Deep-dive reference: For a complete understanding of why the testbed is designed this way — the architecture, core concepts, design philosophy, and key decisions — see the Technical Overview and Design Decisions documents.

    What The Testbed CAN Do¶

    1. Reproduce 5 Historical Bridge Exploits at L2 (Fork-State)¶

    Attack Chain Fork Block Damage What It Proves
    Wormhole Guardian Bypass Ethereum 14,125,553 $326M Forged VAA accepted by TokenBridge via completeTransfer()
    Nomad Replica Fraud Ethereum 15,259,100 $190M process() accepts forged message when committedRoot == 0x00
    Poly Network PrivKey Ethereum 12,996,693 $611M Hash collision in verifyHeaderAndExecuteTx replaces bridge keeper
    Horizon Multisig Drain Ethereum 15,005,526 $100M 3 bridge contracts verified, multisig authorization flow demonstrated
    BNB Token Hub Proof BSC 22,166,659 $570M CrossChain contract identified, IAVL proof bypass mechanism documented

    Each reproduction: - Forks the chain at the correct historical block - Verifies vulnerable contracts exist with non-empty code - Forges and sends exploit calldata - Verifies the exploit succeeded via storage slot changes - Produces deterministic artifact streams (events, gas, storage diffs, balance diffs)

    2. Run L4 Multi-Chain Bridge Demonstrations¶

    Experiment Topology Chains What It Does
    nomad_bridge_l4 nomad_bridge ethereum + polygon Deploys NomadReplica contract on ethereum, sends forged cross-chain message
    demo_agent single ethereum Runs LLM agent workflow with mock provider (no API key needed)

    3. Deploy Bridge Contracts to Forked Chains¶

    Three bridge plugins are available for L4 experiments:

    Bridge Plugin Chains Vulnerability Model
    nomad_bridge ethereum + polygon committedRoot == 0x00 — forged message accepted
    poly_network_bridge ethereum + polygon + bsc Hash collision in _executeCrossChainTx
    wormhole_bridge ethereum + polygon Guardian bypass — forged VAA accepted

    4. Run LLM-Driven Agent Experiments¶

    • Supports OpenAI, Anthropic, Ollama, and custom providers
    • Mock provider included for testing without any API key
    • Agent privilege levels: low, medium, high
    • Configurable task, objective, and account restrictions

    5. Produce Verifiable, Deterministic Artifacts¶

    Every experiment run produces:

    Artifact Format What It Records
    run.jsonl JSONL All lifecycle events (phases, actions, status)
    events.jsonl JSONL All contract events emitted
    storage_diffs.jsonl JSONL Every storage slot that changed (before/after)
    gas_profile.jsonl JSONL Gas used per transaction
    balance_diffs.jsonl JSONL ETH/BNB balance changes
    cross_chain.jsonl JSONL Cross-chain message tracking
    run_manifest.json JSON Full provenance (git commit, Foundry version, config fingerprint)
    log_integrity.json JSON SHA-256 hashes of all streams
    artifact_bundle_manifest.json JSON SHA-256 hashes for export verification

    6. Verify Artifact Integrity¶

    • crucible verify <path> auto-detects run dirs, export bundles, and archives
    • --schemas validates JSONL stream structure
    • SHA-256 hashes detect tampering, missing files, or extra files

    7. Run With or Without Docker¶

    Mode Setup Time Best For
    Docker ~3 min (one-time build) Formal reproductions, publications
    Local (uv + venv) ~1 min (one-time sync) Development, fast iteration
    Local (system-wide) ~1 min Dedicated testbed machines

    What The Testbed CANNOT Do¶

    1. Reproduce Non-EVM Attack Components¶

    Crucible is EVM-only. It uses Foundry's anvil which can only fork EVM-compatible chains (Ethereum, BSC, Polygon). Attacks involving:

    • Solana (Wormhole root cause was Solana-side — EVM-side effects only are reproducible)
    • Cosmos/IBC (cross-chain messages originating from non-EVM chains)
    • Harmony ONE (Horizon bridge — Harmony chain is not EVM)
    • Moonriver (Multichain — Moonriver is Substrate/Kusama-based)
    • Substrate-based chains (Polkadot, Kusama, etc.)

    ...are documented in excluded_components in each attack's evidence.yaml. The non-EVM portion is modeled, not reproduced.

    2. Reproduce Infrastructure Compromises¶

    Attacks that were key compromises, not smart contract bugs:

    Attack Type Why Not Reproducible
    Multichain (July 2023) MPC key shard theft No contract vulnerability; keys were legitimately authorized
    Horizon (June 2022) 2-of-5 multisig key compromise Contracts behaved as designed; reproducing the tx proves nothing

    These are correctly classified as L1 (narrative) — the testbed documents the incident but cannot reproduce it on-chain.

    3. Serve Full Archive State for Polygon/BSC at Old Blocks¶

    This is an RPC provider limitation, not a Crucible bug. At the fork blocks used by the attack packs:

    Chain Fork Block Free Archive? Paid Archive?
    Ethereum 12-16M ✅ Ankr free tier ✅
    BSC 9.7M or 22M ❌ No free provider ✅ GetBlock $39/mo
    Polygon 16.3M ❌ No free provider ✅ GetBlock $39/mo

    Even paid "archive" providers may fail to serve state for non-existent addresses at old fork blocks. This affects anvil_setBalance when bridge plugins try to fund new deployer addresses. The fix (applied to all 3 bridge plugins) wraps per-chain deployment in try/except — the offending chain is skipped gracefully.

    4. Run L4 Experiments Without --no-fidelity¶

    L4 fidelity enforcement checks for: - 2+ chains in historical_scope.chains - Bridge plugin in topology - excluded_components declared

    Bridge demo experiments (e.g., nomad_bridge_l4) don't meet these requirements by design — they're demonstrations, not formal reproductions. Use --no-fidelity for L4 demos.

    5. Guarantee Bit-Identical Cross-Platform Results¶

    The run_manifest.json captures the full provenance (git commit, Foundry version, Python version, config fingerprint), but: - Different Foundry nightly versions may produce different gas estimates - Different RPC providers may return slightly different state roots - Cold vs warm cache affects timing but not results

    For formal publications, use Docker (pinned environment) and run twice with crucible diff.

    Known Issues & Workarounds¶

    Artifact Protection (June 2026)¶

    Artifacts are the most valuable output of the testbed. They are protected by multiple layers:

    Layer 1: clean --all Never Touches Artifacts¶

    crucible clean --all only removes runtime state: .fork_cache, states/checkpoints/, states/snapshots/. It does NOT touch: - logs/ (experiment run artifacts) — protected unless --logs is explicitly passed - datasets/ (exported publication bundles) — protected unless --datasets --confirm is explicitly passed

    This means it's safe to run clean --all between experiments without worrying about losing results.

    Layer 2: Explicit Flags for Artifact Cleanup¶

    Flag What It Cleans Protection
    --logs Run log directories (logs/*/*) saved and published tier runs preserved via run_manifest.json tier check
    --logs-older-than N Logs older than N days Same tier-based protection
    --datasets Exported dataset bundles (datasets/*) Requires --confirm; .protected marker preserved

    Layer 3: Publish Command Adds .protected Marker¶

    Running crucible publish <run_id>: 1. Sets the run's tier to published — protected from --logs and --logs-older-than 2. Writes a .protected marker file in datasets/<run_id>/ — protected from --datasets --confirm

    Layer 4: Docker Persistence¶

    The datasets/ directory is on the bind mount (.:/workspace), not on a Docker named volume. This means: - Export bundles survive container restarts and docker compose down - Files are directly accessible on the host filesystem at ./datasets/ - Not affected by Docker volume pruning

    Layer 5: Export Idempotency¶

    crucible export <run_id> will NOT overwrite an existing export bundle unless --force is passed. If the bundle already exists, it re-verifies integrity and regenerates reports without touching the copied source files.

    Known Issues & Workarounds¶

    Issue Affects Workaround
    Polygon archive at block 16,332,982 can't serve new addresses poly_network_bridge, nomad_bridge, wormhole_bridge Chain skipped gracefully; use GetBlock or QuickNode for full support
    BSC archive at block 9,755,172 needs paid provider poly_network_replay (3-chain), bnb_token_hub_replay Use GetBlock Starter ($39/mo) or run Ethereum-only
    Auto-merge may kill + restart chains between experiments All matrix runs Use --no-auto-merge to bypass; or let it re-fork automatically (fixed)
    Port TIME_WAIT after killing Anvil Auto-merge restart Crucible polls port for 10s before restarting (fixed); wait 30s if still failing
    Checkpoints snapshot dead chains Topology switching rm -f states/*.meta.json before switching (fixed in framework)
    Bridge bytecode double 0x prefix New bridge plugin development Use "data": bytecode (not "0x" + bytecode) — Foundry already has 0x
    Multichain L1 doesn't return AttackResult multichain_replay in benchmark matrix Use --continue-on-error when running matrix
    Agent experiments need LLM API key demo_agent with real provider Use agent_provider: mock for testing without API key

    What's Been Fixed (June 2026)¶

    23 issues found and resolved across 3 validation rounds (22 bugs + 1 enhancement). See docs/REPRODUCIBILITY.md for the full log.

    Framework (4 bugs fixed)¶

    • Auto-merge kills chain but never restarts → now re-forks automatically
    • Port TIME_WAIT after killing old Anvil → port polling loop (500ms × 20 attempts)
    • Checkpoints snapshot dead chains → filtered to active topology only
    • Fidelity failures hidden from console → reasons printed inline below [fail]

    Bridge Plugins (3 bugs fixed)¶

    • Double 0x prefix in bytecode → removed extra "0x" +
    • Precompile deployer fails on archive nodes → try/except per-chain
    • Placeholder string in bridge evidence.yaml → empty list []

    CLI (1 feature added)¶

    • --no-auto-merge flag to bypass automatic fork block merging

    Determinism & Reproducibility Verification¶

    Determinism Checks (Run Twice + Diff)¶

    All 5 benchmark experiments were run twice and diffed to verify deterministic reproduction:

    Experiment Topology Chains storage_equal events_equal gas_equal balances_equal Result
    wormhole_replay bridge ethereum + polygon ✅ true ✅ true ✅ true ✅ true Deterministic
    nomad_replay bridge ethereum + polygon ✅ true ✅ true ✅ true ✅ true Deterministic
    horizon_replay single ethereum ✅ true ✅ true ✅ true ✅ true Deterministic
    bnb_token_hub_replay single_bsc bsc ✅ true ✅ true ✅ true ✅ true Deterministic
    poly_network_replay poly_network_full ethereum + polygon + bsc ❌ false ✅ true ❌ false ❌ false Non-deterministic

    4 of 5 experiments are fully deterministic. The only non-deterministic result (poly_network) is expected — it forks 3 chains simultaneously and Polygon/BSC archive RPCs may return slightly different state per run (see Archive RPC Limitations).

    Cross-Provider Verification¶

    Not run. The .env includes only one free-tier RPC provider per chain:

    ETHEREUM_RPC_URL=https://rpc.ankr.com/eth
POLYGON_RPC_URL=https://rpc.ankr.com/polygon
BSC_RPC_URL=https://rpc.ankr.com/bsc

    A second RPC provider (e.g., GetBlock, QuickNode, Alchemy) would be needed to confirm results are provider-independent. At the code level, the framework has no provider-dependent logic — all chain interaction goes through the JSON-RPC client abstraction. Any provider-capable client should produce equivalent results if they serve the same archive state.

    Export Bundles¶

    4 of 5 experiments have publication-ready export bundles in datasets/:

    Experiment Run ID Path Files Integrity
    wormhole_replay DS-2026-00155 datasets/DS-2026-00155 24 ✅ PASS
    nomad_replay DS-2026-00158 datasets/DS-2026-00158 24 ✅ PASS
    horizon_replay DS-2026-00160 datasets/DS-2026-00160 18 ✅ PASS
    bnb_token_hub_replay DS-2026-00159 datasets/DS-2026-00159 18 ✅ PASS

    File count varies by topology complexity: bridge-topology experiments (wormhole, nomad) produce 24 files from 2 forked chains; single-chain experiments (horizon, bnb) produce 18 files from 1 chain.

    Each bundle contains: - artifact_bundle_manifest.json — SHA-256 hashes of every file - publication_report.json — machine-readable report with outcome, assertions - provenance.json — extracted provenance data - assertions.json — all assertion results from the run - REPRODUCE.md — exact reproduction instructions - All original run artifacts (run.jsonl, events.jsonl, storage_diffs.jsonl, etc.)
