    Instrumentation Architecture — RPC Proxy Design¶

    Overview¶

    The instrumentation layer is an RPC proxy that sits transparently between any code that talks to the local Anvil fork (attack packs, Foundry scripts, LLM agents, manual cast send calls) and the Anvil process itself.

    Every transaction is automatically traced — the attack pack does zero artifact collection work.

    Blank Fork Mode — Zero Remote RPC Dependency¶

    The diagrams above show the archive fork path: Anvil fetches historical state from a remote archive RPC provider. However, the instrumentation proxy is completely agnostic to how Anvil was started — it only cares that Anvil is listening on an HTTP port.

    How Blank Forks Work¶

    In blank fork mode (fork_block: null), Anvil starts without --fork-url:

    # Archive fork: Anvil fetches remote state
anvil --fork-url https://eth.llamarpc.com --fork-block-number 15259100 --accounts 0 --balance 0

# Blank fork: No remote dependency — Anvil is a standalone local EVM
anvil --accounts 0 --balance 0

    The proxy forwards to whatever URL it's given — it does not inspect or care about the upstream chain state. The same RpcProxy class, the same attach_instrumentation() call, the same ThreadPoolExecutor background tracing. No code changes, no config changes.

    Why This Matters¶

    Property Archive Fork Blank Fork
    Remote RPC needed? Yes — archive provider No — fully offline
    Startup time 30s–5 min (cold cache) <1s — instant
    Determinism Depends on archive state Fully deterministic
    CI/hermetic testable Needs fixture replay Yes — directly
    Artifact collection Same proxy + streams Same proxy + streams
    Fidelity levels L2 (verified fork state) L3/L4 (validation, no history)

    The artifact collection pipeline is identical in both modes. The instrumented data — gas, events, storage diffs, balance diffs — has the same format, the same integrity hashing, the same diffability. The only difference is whether Anvil fetches historical state at startup.

    Practical Use Cases¶

    • L4 validation experiments (l4_full_validation, nomad_bridge_l4): Two blank forks (ethereum + polygon), bridge plugin deployed on both, bidirectional cross-chain messaging. Zero RPC calls during the entire run — the proxy only talks to local Anvil instances.
    • Offline demo/testing: Run crucible experiment run with blank forks on a plane, in CI, or on an air-gapped machine. Artifact collection works identically.
    • Development workflow: Iterate on attack packs without waiting for archive forks. Switch to archive forks only when ready for L2 fidelity verification.

    Important caveat: Blank forks can fail on first startup due to an Anvil nightly-build quirk with --accounts 0 --balance 0. See Troubleshooting §7C and Testbed Scope Guide — Blank Fork Caveats for recovery steps.

    ER->>AK: run(context)

    AK->>PX: eth_sendTransaction({...})
    PX->>AN: forward eth_sendTransaction
    AN-->>PX: tx_hash
    PX-->>AK: tx_hash

    par Background Trace (via ThreadPoolExecutor)
        PX->>AN: poll eth_getTransactionReceipt(tx_hash)
        AN-->>PX: receipt {gasUsed, logs, status}
        PX->>AN: debug_traceTransaction(tx_hash, prestateTracer + diffMode)
        AN-->>PX: {pre: {accounts}, post: {accounts}}
        PX->>LE: gas(step="auto_tx", gas_used=21000)
        PX->>LE: event(chain, event_name="Transfer", ...)

    AK-->>ER: AttackResult(success=true)

    CM->>PX: stop() — shutdown HTTP server

    ER->>LE: write_report(manifest_path)
    ER->>LE: write_integrity() — hashes + closes handles

    Integration Points¶

    Component Role File
    RpcProxy HTTP proxy + background tracer (gas, events, storage diffs, balance) crucible/instrumentation.py
    ChainManager Starts/stops proxy per chain; runtime_for() returns proxy URL transparently crucible/chain_manager.py
    ExperimentRunner Starts proxy before attack, drains/stops in finally block crucible/experiment.py
    LogEmitter Receives traced artifacts into typed JSONL streams crucible/logging.py
    diff.py Compares artifact streams across runs for determinism check crucible/diff.py

    Proxy URL Transparency¶

    # Before instrumentation:
runtime_for("ethereum").rpc_url  # → "http://127.0.0.1:8545"

# After start_instrumentation("ethereum", log):
runtime_for("ethereum").rpc_url  # → "http://127.0.0.1:54321" (proxy)

    All existing code continues to work unchanged — the proxy is invisible to attack packs, scenario steps, and bridge plugins.

    Standalone Mode (crucible instrument)¶

    Beyond experiment-mode auto-instrumentation, crucible instrument starts the proxy standalone — no fork, no experiment runner, no scenario. This is useful for ad-hoc research where you:

    1. Start an Anvil fork manually (or use an existing one)
    2. Point the proxy at it with --upstream
    3. Send transactions through the proxy URL (via cast send, Foundry scripts, Python scripts)
    4. Get complete artifact streams without writing any instrumentation code
    # Terminal 1: Start Anvil manually
anvil --fork-url https://eth.llamarpc.com --fork-block-number 15259100

# Terminal 2: Start the instrumentation proxy
crucible instrument --upstream http://127.0.0.1:8545 --chain ethereum --output-dir ./logs/my_run

# Terminal 3: Send transactions through the proxy
cast send --rpc-url http://127.0.0.1:<proxy-port> 0xTarget --value 1ether

    Standalone Options¶

    Flag Default Description
    --upstream, -u http://127.0.0.1:8545 Upstream RPC URL to proxy (typically the Anvil fork)
    --chain, -c ethereum Chain name for artifact tagging and subdirectory naming
    --output-dir, -o ./instrument_output Directory to write artifact streams into
    --port, -p 0 (random) Proxy listen port. Use a specific port for scripting/chaining
    --timeout, -t 0 (run until Ctrl+C) Auto-exit after N seconds. Useful for scripted runs

    Artifacts are written to --output-dir with the same JSONL stream structure as experiment-mode captures.

    Thread Safety¶

    Concern Solution
    Multi-chain topologies _make_handler() factory creates a fresh handler class per proxy instance — each chain's proxy forwards to its own Anvil independently
    Rapid-fire tx loops ThreadPoolExecutor(max_workers=10) bounds concurrent traces; excess txs queue
    Race with LogEmitter.close() drain() → stop() ordering: executor shutdown waits for pending traces, then HTTP server stops, then write_integrity() closes handles

    Fallback Behavior¶

    When debug_traceTransaction is unavailable or returns an error:

    Artifact Captured? Source
    Gas used [ok] Yes eth_getTransactionReceipt
    Events / logs [ok] Yes eth_getTransactionReceipt
    Storage diffs [fail] Warning logged N/A — no trace available
    Balance changes [fail] Warning logged N/A — no trace available

    A prominent failure step is logged showing the tx hash, what was lost (storage diffs + balance changes), what was saved (gas + events), and a reference to TROUBLESHOOTING.md for Anvil version advice.
