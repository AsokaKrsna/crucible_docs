    Item 6: crucible verify Command¶

    Effort: M

    New crucible verify <path> command that auto-detects run vs bundle directories and re-computes SHA-256 hashes against integrity manifests.

    • Files: crucible/export.py (added verify_run(), verify()), crucible/commands/verify.py (new)
    • Detection: log_integrity.json = run directory, artifact_bundle_manifest.json = bundle, both = dual-pass
    • Reports: Hash mismatches, missing files, extra files (informational only)
    • Exit code: Non-zero on verification failure
    • Status: [ok] Complete

    3. Phase 4 — Publication & Validation¶

    Item 12: Per-Attack Fidelity Table (BENCHMARKS.md)¶

    Effort: XS

    Added a "Per-Attack Fidelity Assessment" section to docs/BENCHMARKS.md with a table showing all 6 benchmark packs, their declared fidelity levels, upper bounds, and specific constraints.

    • Table covers: nomad_replica_fraud (L2, ceiling L3), wormhole_guardian_bypass (L2, ceiling L2), poly_network_privkey (L2, ceiling L4), horizon_multisig_drain (L2, ceiling L2), multichain_router_permit (L1, ceiling —), bnb_token_hub_proof (L2, ceiling L3)
    • Also added: Fidelity Level Legend (L0-L4 definitions)
    • Status: [ok] Complete

    Item 13: Run Tiers (scratch/saved/published)¶

    Effort: M

    Added run retention tiers with auto-cleanup, plus crucible publish command.

    • Files: crucible/config.py (RUN_TIERS type, CleanupConfig fields), crucible/provenance.py (run_tier in manifest), crucible/history.py (run_tier DB column), crucible/experiment.py (run_tier param), crucible/commands/experiment.py (--tier flag), crucible/commands/export.py (publish command), crucible/cleanup.py (tier-aware removal)
    • Tiers: scratch (auto-clean 7 days), saved (indefinite), published (indefinite + prominent)
    • Usage: crucible experiment run ... --tier saved | crucible publish <run_id>
    • Tests: 15 new tests in tests/unit/test_run_tiers.py
    • Status: [ok] Complete

    Item 14: JSONL Schema Validation¶

    Effort: S

    Bundle-time validation of all 6 JSONL artifact streams against expected schemas.

    • File: crucible/schema.py (new) — defines required/optional fields and type constraints for each stream
    • Integration: crucible verify --schemas flag, ExportBundle.create() logs warnings
    • Streams validated: run.jsonl, events.jsonl, storage_diffs.jsonl, gas_profile.jsonl, balance_diffs.jsonl, cross_chain.jsonl
    • Scans: Root-level + chain_* + bridge/ subdirectories automatically
    • Tests: 22 new tests in tests/unit/test_schema.py
    • Status: [ok] Complete

    Item 15: --from-tx Attack Scaffolder¶

    Effort: L

    Added --from-tx option to crucible attack init that fetches on-chain transaction metadata and populates the scaffold with real data.

    • File: crucible/commands/attack.py
    • New helper: _fetch_tx_metadata() — connects to RPC, extracts fork block, contract address, attacker address, calldata preview
    • Graceful degradation: Returns defaults (no crash) when RPC unavailable
    • Populates: evidence.yaml (fork_block, exploit_transactions, affected_contracts), plugin.py (CONTRACT_ADDRESS constant, get_code check, calldata preview), scenario.yaml (fork_blocks)
    • Usage: crucible attack init bnb_token_hub --chain bsc --from-tx 0x5ea16a... --rpc-url https://...
    • Tests: 16 new tests in tests/unit/test_attack_from_tx.py
    • Status: [ok] Complete

    Item 16: Fidelity Level Runtime Enforcement¶

    Effort: M

    Runtime validation that experiment behavior matches the fidelity level declared in evidence.yaml.

    • File: crucible/enforcer.py (new) — FidelityEnforcer class with level-scoped checks
    • Enforcement per level:
    • L0: None (always passes)
    • L1: Attack plugin returned a result
    • L2: Evidence chains in topology, trace artifacts present, no trace failures
    • L3: exact_tx_replay=true, exploit_transactions non-empty, tx actions in run.jsonl
    • L4: Multi-chain topology + bridge plugin + excluded_components declared
    • Integration: Post-run check in ExperimentRunner.run(), guarded by outcome == "success"
    • Flags: Fidelity enforcement enabled by default; use --no-fidelity to skip. publication.fidelity_enforcement (config) falls back to publication gates.
    • Tests: 19 new tests in tests/unit/test_enforcer.py
    • Status: [ok] Complete

    Item 17: Bridge Plugin Evidence Requirements¶

    • New file: bridges/demo_bridge/evidence.yaml — complete bridge evidence
    • Tests: 10 new tests in tests/unit/test_bridge_evidence.py

    5. Known Gaps & Cosmetic Issues¶

    1. Docker volume artifact access — Artifacts produced by Docker experiments live inside Docker named volumes (logs_data, states_data), not directly on the host filesystem. Use docker compose run --rm orchestrator ls logs/... to inspect artifacts from Docker runs.

    2. CLI_REFERENCE.md Maintenance — The reference doc should be regenerated when commands change. Currently maintained manually.

    3. Local Foundry requirement — On systems without Foundry binaries on PATH (common on Windows), all experiments must run through Docker. The crucible doctor command correctly reports this condition.
