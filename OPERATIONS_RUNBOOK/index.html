    Operations Runbook¶

    Testing¶

    Run unit tests and smoke tests locally (no external services needed):

    # Unit tests
docker compose run --rm orchestrator bash -c "uv run pytest tests/unit/ -v"

# Full smoke test (from repo root)
docker compose build
docker compose run --rm orchestrator init
docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/demo_experiment.yaml --no-baseline
"
docker compose run --rm orchestrator logs list
docker compose run --rm orchestrator history --rebuild

    Note: CI pipelines have been removed — this is a local-only project. All testing is done on-device.

    Managing Fork Blocks Per Experiment¶

    Each attack pack requires a specific fork block on Ethereum (and possibly other chains). testbed.yaml has a single fork_block per chain, so you need testbed.local.yaml (gitignored) to switch between experiments:

    # testbed.local.yaml — deep-merged with testbed.yaml
chains:
  ethereum:
    fork_block: 15259100  # Nomad Bridge fork
    Attack Ethereum Polygon BSC
    Nomad Bridge 15,259,100 — —
    Wormhole Guardian 14,125,553 — —
    Poly Network 12,996,693 16,332,982 9,755,172
    Horizon Multisig 14,399,491 — —
    BNB Token Hub — — 22,166,659
    Multichain Router 16,329,241 — —

    Workflow:

    # 1. Edit testbed.local.yaml with the fork block for your target experiment
# 2. Fork the chain with the correct topology
crucible fork --topology <name>

# 3. If the fork block is wrong (contract has no code), Crucible's EOA check
#    will catch it with a clear error message via `crucible evidence check`

    Running Determinism Checks¶

    Proving an experiment is deterministic is a two-run workflow:

    # Run 1:
docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"
# Note run ID A (e.g., DS-2026-00001)

# Run 2:
docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"
# Note run ID B (e.g., DS-2026-00002)

# Compare:
docker compose run --rm orchestrator diff DS-2026-00001 DS-2026-00002

    Expected output: all 4 metrics true (storage, events, gas, balances).

    Daily Start¶

    crucible status --disk
crucible plugin list
crucible abi validate
crucible cache inspect

    Check for:

    • orphaned chain metadata
    • large state/cache growth
    • invalid plugins
    • invalid ABI JSON
    • cache entries with stale provider/block identity

    Start A Research Session¶

    crucible fork --topology <topology>
crucible baseline verify <scenario>

    If no baseline exists:

    crucible baseline capture <scenario> --topology <topology>

    L4 Research Session (With Bridge Plugin)¶

    # 1. Validate the bridge plugin
crucible plugin list --type bridge
crucible plugin validate bridges/nomad_bridge

# 2. Fork with bridge topology (deploys contracts to all chains automatically)
crucible fork --topology nomad_bridge

# 3. Run experiment with fidelity enforcement
crucible experiment run experiments/my_l4_experiment.yaml

# 4. Check cross-chain artifacts
crucible logs list
cat logs/<experiment>/<run_id>/bridge/cross_chain.jsonl

    Available bridge topologies: - nomad_bridge — ethereum + polygon (Nomad Replica) - poly_network_bridge — ethereum + polygon + bsc (EthCrossChainManager/Data) - wormhole_bridge — ethereum + polygon (WormholeCore/TokenBridge)

    Reset Between Runs¶

    crucible reset --scenario <scenario>

    Stop Everything¶

    crucible clean

    Cleanup Disk¶

    crucible clean --checkpoints
crucible clean --cache
crucible clean --logs-older-than 30
crucible status --disk

    Rebuild History¶

    crucible history --rebuild
crucible history --limit 20
crucible history --tag wormhole --outcome success  # filter by tag + outcome

    Auto-Extract Attack Packs¶

    # From a known exploit transaction hash
crucible attack extract <tx_hash> --name my_attack --chain ethereum

# Validate the generated pack
crucible plugin validate attacks/my_attack

    Export & Verify Bundles¶

    crucible export <run_id> --format tar-gz
crucible bundle hash datasets/<run_id>
crucible verify datasets/<run_id>

    Compare Run Results¶

    crucible diff DS-2026-00001 DS-2026-00002

    Safe Previews¶

    crucible --dry-run clean --all
crucible --dry-run reset --scenario demo_scenario --mode hard

    Troubleshooting¶

    Port In Use (Orphaned Processes)¶

    ERROR: port 8545 is already in use for chain ethereum

    Cause: A previous crucible fork or manual anvil instance is still running. Each uv run crucible fork spawns a process tree that can persist as orphans: 

    uv run crucible fork --topology <name>
  └── crucible fork --topology <name>
       └── anvil --port 8545 ...
    When the parent uv run is killed (by timeout, crash, or Ctrl+C), the children may survive.

    Recovery steps:

    # 1. Find what's on the port
fuser -v 8545/tcp
# or: ss -tlnp | grep 8545

# 2. Kill the entire process tree
#    Kill the anvil child first, then the crucible fork parent
kill -9 <anvil_PID>
kill -9 <crucible_fork_PID>

# 3. OR nuke everything at once (faster)
pkill -9 -f 'anvil' 2>/dev/null
pkill -9 -f 'crucible fork' 2>/dev/null
sleep 2

# 4. Remove stale meta files
rm -f states/*.meta.json

# 5. Verify all ports free
ss -tlnp | grep -E '854[5-8]' || echo 'All free'

    If it keeps happening: Check for zombie processes, tmux sessions, or Docker containers managing the process.

    Port override (multi-researcher environments): If another researcher legitimately owns the port, override it in testbed.local.yaml:

    chains:
  ethereum:
    port: 8555  # use a different port

    Prevention: Always run crucible clean --all between experiments, and use consistent port configuration.

    Zombie (Defunct) Processes¶

    # ps aux shows: [anvil] <defunct> with state Z

    Cause: A zombie process has exited but hasn't been reaped by its parent. Zombies can't be killed (SIGKILL has no effect) and don't hold ports.

    Detection: 

    ps aux | grep defunct | grep anvil

    Handling: Crucible's chain_manager now detects zombies via /proc/<pid>/status (Linux) and treats them as already terminated. They will be reaped by init (PID 1) when their parent dies.

    Port Keeps Getting Reoccupied After Kill¶

    Symptom: You free the port, verify it's free, but the next fork command says it's occupied again.

    Likely causes: 1. Process tree — Killing anvil doesn't kill the parent crucible fork (which may restart it) 2. Previous fork still running — A uv run crucible fork from an earlier command is still alive 3. Not waiting long enough — The port may be in TIME_WAIT state (30-120s after kill)

    Fix: Use the full cleanup procedure above (pkill + remove meta + verify). For TIME_WAIT, wait 30s or use a different port.

    Baseline Mismatch¶

    Do not overwrite the baseline immediately. First identify why the state changed:

    crucible baseline verify <scenario>

    If the baseline should change, capture a new one and document the reason.

    RPC Too Slow Or Expensive¶

    Use cached fork mode (fork_cache: { cache_policy: strict }) or local archive mode. Run crucible cache inspect to check cache state.

    Invalid Plugin¶

    crucible plugin validate attacks/<name>

    Fix manifest, evidence, entry file, or class inheritance issues.

    Docker: "sh -c" Command Fails¶

    $ docker compose run --rm orchestrator sh -c "..."
Fails with: crucible: error: argument command: invalid choice: 'sh'

    Cause: The orchestrator's Docker entrypoint is ["uv", "run", "crucible"], not a shell. Running sh -c passes sh as a subcommand to crucible.

    Fix: Override the entrypoint with --entrypoint bash:

    docker compose run --rm --entrypoint bash orchestrator -c "
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"

    Docker: Stale Meta File Errors¶

    ERROR: RPC call failed for anvil_dumpState at http://127.0.0.1:8547: Connection refused

    Cause: The states/ Docker volume persists .meta.json files from previous forks. When you switch topologies (e.g., from poly_network back to single), stale meta files reference chains that aren't running.

    Fix: Always clean stale meta files before forking:

    docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single
"

    Docker: Chain Not Ready (Connection Refused)¶

    ERROR: chain bsc did not become ready within 30s on http://127.0.0.1:8547: Connection refused

    Cause 1: The RPC URL in testbed.yaml is an environment variable (${BSC_RPC_URL}) that isn't set.

    Fix: Check the env var: 

    docker compose run --rm --entrypoint bash orchestrator -c 'echo "BSC_RPC_URL=$BSC_RPC_URL"'

    If empty, set it in .env.

    Cause 2: The RPC provider doesn't support archive queries at the configured fork block.

    Fix: Test archive depth: 

    docker compose run --rm --entrypoint bash orchestrator -c '\
  curl -s -X POST -H "Content-Type: application/json" \
  -d "{\"jsonrpc\":\"2.0\",\"id\":1,\"method\":\"eth_getBlockByNumber\",\"params\":[\"0x94E334\",false]}" \
  https://your-rpc-provider\
'

    See docs/RPC_PROVIDER_GUIDE.md for archive-compatible providers.

    Fork Block Wrong For Experiment¶

    Attack verification failed: EthCrossChainManager has no code at fork block 12996062

    Cause: The testbed.yaml fork_block is set for a different experiment (e.g., Wormhole instead of Poly Network).

    Fix: Check which fork block each experiment needs and update testbed.yaml or create a testbed.local.yaml override:

    # testbed.local.yaml
chains:
  ethereum:
    fork_block: 12996693  # Poly Network fork
    Attack Ethereum Fork Block
    Nomad Bridge 15259100
    Wormhole Guardian 14125553
    Poly Network 12996693

    Contract Address Verification¶

    Attack verification failed: ContractName has no code at fork block

    Cause: The contract address in the attack pack might be wrong — several attack packs had addresses that were actually EOAs, not contracts.

    Fix: Always verify addresses on Etherscan before using them:

    1. Go to Etherscan
    2. Paste the address
    3. Verify it shows "Contract" (not "Address (EOA)")
    4. Check the contract has non-zero transaction history

    Foundry Not Available¶

    crucible: error: anvil not found on PATH

    Inside Docker: Foundry is pre-installed. If missing, rebuild: 

    docker compose build

    Locally: Install Foundry: 

    curl -L https://foundry.paradigm.xyz | bash
foundryup
