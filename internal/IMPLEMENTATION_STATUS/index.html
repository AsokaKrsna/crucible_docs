    • topology and testbed config parsing with Pydantic validation

    • deterministic Anvil command generation with gas, fee, order, timestamp controls
    • Anvil process lifecycle management (start, stop, status, health check polling)
    • RPC provider fallback system with priority-based failover and token-bucket rate limiting
    • RPC budget enforcement — per-run request budget with configurable max_requests_per_run and fail_if_exceeded
    • snapshot and baseline RPC wrappers (dumpState / loadState)
    • baseline manifest capture, fingerprint verification (SHA-256), and restoration
    • structured JSONL log streams (run, events, storage diffs, gas profile, cross-chain)
    • log integrity hashing and auto-generated report.md
    • plugin SDK with AttackPlugin, BridgePlugin, CruciblePlugin interfaces
    • plugin registry with discovery, manifest validation, and dynamic loading
    • evidence manifest validation with fidelity level enforcement
    • scenario engine with YAML atomic steps, forge_script, and python_script execution
    • formal experiment runner with manifest generation, artifact export, and matrix runner
    • 3-tier reset system: soft (anvil_loadState), medium (kill+relaunch+restore), hard (re-fork+re-bootstrap)
    • phase-level checkpointing and experiment resume with phase-skipping
    • SQLite history index with tag/outcome filtering
    • canonical run diff (storage, events, gas comparison)
    • RPC/cache estimator
    • ABI registry listing, validation, and fetching (Etherscan/Sourcify)
    • bootstrap.py execution in setup workflow
    • --dry-run global flag for safe previews
    • runtime cleanup commands (cache, checkpoints, logs, states, orphans)
    • dataset versioning (DS-YYYY-NNNNN monotonic IDs)
    • canonical bootstrap.py template for pre-attack state preparation
    • comprehensive __init__.py public API
    • crucible doctor — environment validation command covering Python imports, Foundry, config, RPC resolution, directories, Streamlit, Docker
    • Environment validation via crucible doctor (new)
    • RPC budget enforcement — per-run request budget via RpcProviderManager (removed from class-level TokenBucket state)
    • Bridge lifecycle integration — deploy/configure/connect before scenario, destroy after attack (ordering fix)
    • Bridge destroy in finally blocks — bridge_plugin.destroy() moved to finally block in both run() and resume() so bridge state is always cleaned up (even on exception)
    • RPC provider fallback — 4 fallback providers per chain (LlamaRPC, Ankr, PublicNode) configured in testbed.yaml
    • DeterminismConfig — removed deprecated coinbase field; backward-compat via extra = "ignore"
    • user, maintenance, attack-pack, benchmark, research, operations, handoff, status docs- user, maintenance, attack-pack, benchmark, research, operations, handoff, status docs

    Agent System Layer [ok] (Phase 4–6 Complete)¶

    • AgentPlugin ABC — observe → think → act lifecycle with privilege enforcement
    • AgentContext — wraps ExperimentContext with tool history, audit, and safe parameter filtering
    • 9 agent tools: chain_id, get_balance, get_code, get_storage, call_contract, get_logs (low), impersonate, send_transaction (medium), advance_time (high)
    • 3 privilege levels: low (read-only), medium (impersonate+send), high (full access)
    • Tool-based security — parameter filtering against tool schema, privilege gating on every call
    • verify_tracing scenario step — post-capture verification of instrumentation artifact streams
    • bridge_action scenario step — bridge operations during scenarios (send_message, relay_messages, check_delivery, verify_state)
    • LLMProvider ABC — abstract base for LLM provider implementations
    • OpenAIProvider — raw urllib-based OpenAI-compatible provider with tool calling
    • AnthropicProvider — raw urllib-based Anthropic Messages API with tool use
    • create_provider() factory — openai, anthropic, ollama, local, custom provider types
    • Ollama/local endpoint support — auto-detects localhost, no API key required for local endpoints
    • LLMAgentPlugin — concrete AgentPlugin using LLM for autonomous decision-making
    • System prompt builder — constructs prompts with tool descriptions, task, objective, constraints
    • Response parser — handles OpenAI function_calls, Anthropic tool_use, and done JSON signal
    • Conversation management — history truncation at 50 messages, retry on transient failures
    • Run-config integration — provider type, model, objective, task configured via experiment YAML run block
    • tool_def_to_openai() — converts ToolDef to OpenAI tool format with proper parameter schemas
    • crucible attack extract — auto-generate attack packs from on-chain transactions (fetches tx + ABI + decodes function call)
    • ExportBundle — publication-ready artifact bundles with manifests, reports, provenance, assertions, integrity hashing
    • Signing — bundle hash computation and verification (SHA-256 aggregate + per-file)
    • CacheManager — fork cache fingerprinting, provenance verification, strict/permissive policy enforcement
    • PublicationGate — 6 research-quality gates (evidence completeness, baseline, Foundry version, provenance, etc.)
    • Comprehensive crucible doctor — validates Python, Foundry, config, RPC, ports, Streamlit, Docker
