    Crucible — Developer Handoff Summary¶

    For: SOM and Team Review Date: June 7, 2026 Context: Full A-Z validation, bug fixing, documentation overhaul, hybrid approach testing, and architecture/code quality review resolution

    Executive Summary: Crucible has undergone complete A-Z validation of 3/6 attack packs (Nomad, Wormhole, Poly Network), all proven deterministic across two runs with identical gas, storage, events, and balances. 13 bugs were found and fixed — 4 of which were incorrect contract addresses in the original research data (contract addresses that were actually EOAs). A new Solidity+Python hybrid approach was invented and validated, giving researchers the choice of writing exploit logic in either language. The platform is ~98% complete with 461 passing unit tests. Documentation is comprehensive (25 docs).

    1.3 Solidity + Python Hybrid Approach — Validated¶

    Discovered that forge script --broadcast cannot execute certain contract calls (Foundry's local EVM simulation differs from Anvil's actual execution). Created and validated a hybrid approach:

    Solidity script (ABI-encode calldata) → console.log JSON → 
Python plugin (run forge subprocess, parse JSON, send tx via client.send_transaction())

    4. Two Approaches For Writing Exploit Code¶

    Pure Python (existing working plugins)¶

    # Example: attacks/nomad_replica_fraud/nomad_replica_fraud.py
class NomadReplicaFraud(AttackPlugin):
    def execute(self, context, phase=None):
        client = JsonRpcClient(rpc_url)
        client.impersonate_account(ATTACKER)
        calldata = _encode_process_calldata(forged_message, forged_proof)
        tx_hash = client.send_transaction({...})
        return AttackResult(success=True, phase_results=[...])

    Best for: Complex ABI encoding, dynamic calldata, hash collisions

    Solidity + Python Hybrid (new, validated)¶

    // Exploit.s.sol — computes calldata, doesn't broadcast
contract NomadCalldata is Script {
    function run() external {
        bytes memory calldata_ = abi.encodeWithSelector(selector, arg1, arg2);
        console.log("CALLLDATA_JSON:{\"data\":\"%s\"}", vm.toString(calldata_));
    }
}

    # attack_plugin.py — runs forge as subprocess, parses output, sends tx
subprocess.run(["forge", "script", "Exploit.s.sol:Contract", "--rpc-url", rpc_url])
tx = json.loads(re.search(r"CALLLDATA_JSON:(\{.*\})", output).group(1))
client.send_transaction({"from": tx["from"], "to": tx["to"], "data": tx["data"]})

    Best for: Solidity-native researchers, cleaner ABI encoding, academia

    5. Known Limitations¶

    Limitation Impact
    BSC/Polygon archive RPCs require payment BNB Token Hub + multi-chain Poly Network blocked on free tier
    Daemon Anvil containers have fork block drift risk Use single-container mode instead
    forge script --broadcast simulation differs from Anvil execution Hybrid approach (calldata via forge, tx via RPC) works around this
    No CI pipeline for nightly regression tests Manual validation needed for each change
