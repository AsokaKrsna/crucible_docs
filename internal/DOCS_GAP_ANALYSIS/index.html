    CLI Gaps to Fix¶

    1. crucible history — Queries the SQLite history index (crucible.db). Supports --tag, --outcome, --tier, --limit, --rebuild flags. Shows experiment history with run IDs, outcomes, tiers, and timestamps.
    2. crucible summary — One-shot workspace overview: active chains, recent runs (10), disk usage for logs/states/datasets/fork_cache.
    3. crucible logs list — Lists all run manifests in the logs directory with run ID, experiment name, and outcome.
    4. crucible metadata <run_id> — Displays the run manifest as formatted JSON.
    5. crucible evidence check <attack> --fork --topology — Pre-flight validation of evidence.yaml claims against a running fork. Checks fork block existence, contract code, and emits warnings/errors.
    6. experiment matrix — Full documentation of matrix YAML structure, --continue-on-error, --no-baseline.

    5. Critical Gaps — Ranked by Research Impact¶

    🔴 GAP 1: Error Hierarchy¶

    File: crucible/errors.py Impact: Researchers encounter errors constantly. Without understanding the error hierarchy, they can't distinguish "expected failure" from "something is broken." Has: 11 error types with associated exit codes (CONFIG_INVALID=2, BASELINE_MISMATCH=3, ENVIRONMENT_INVALID=5, RPC_UNAVAILABLE=6, ASSERTION_FAILED=7, PLUGIN_INVALID=8, PUBLICATION_GATE_FAILED=9, EVIDENCE_INCOMPLETE=10, PROVENANCE_INCOMPLETE=11, INTERRUPTED=130).

    🔴 GAP 2: Analysis API¶

    File: crucible/analysis.py Impact: The entire programmatic analysis workflow is undocumented. Researchers who want to load experiment data into Jupyter/Pandas have no reference. Has: find_run(), load_run(), list_runs(), load_events(), load_gas(), load_storage_diffs(), load_balance_diffs(), load_cross_chain(), load_agent_reasoning().

    🔴 GAP 3: Baseline System¶

    File: crucible/baseline.py Impact: Baselines are the foundation of reproducibility. The system is barely documented. Has: BaselineManager with capture(), verify(), restore(), load_manifest(). Dual verification (file integrity + live chain state).

    🔴 GAP 4: Run Tiers & Publication Gates¶

    Files: crucible/config.py (RUN_TIERS, PublicationConfig), crucible/publication.py Impact: The entire research publication workflow (scratch → saved → published) is barely documented. Has: 3-tier system (scratch/saved/published), PublicationGate with fidelity, provenance, and baseline checks.

    🔴 GAP 5: Cache System¶

    File: crucible/cache.py Impact: Fork cache fingerprinting affects determinism claims. Undocumented. Has: CacheManager with build/inspect/verify/invalidate, strict vs permissive policy, manifest fingerprinting.

    🔴 GAP 6: IDs / Run ID Format¶

    File: crucible/ids.py Impact: Every run is identified by a DS-YYYY-NNNNN ID. Format undocumented. Has: next_dataset_id() with year-resetting counter, persisted in states/dataset_counter.yaml.

    🔴 GAP 7: Checkpoint System¶

    File: crucible/checkpoint.py Impact: Experiment resume depends on checkpoints. Undocumented. Has: CheckpointManager, CheckpointIntegrityError. Checkpoints saved/restored per phase.

    🔴 GAP 8: History System¶

    File: crucible/history.py Impact: crucible history and crucible summary use this. Undocumented. Has: HistoryIndex with SQLite-backed query, rebuild from logs, tag/outcome/tier filtering.
