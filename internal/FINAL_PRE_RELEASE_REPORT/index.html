    6. Bugs Found¶

    Bug #1: Evidence validation fails on dict-format affected_contracts (🔧 FIXED)¶

    Location: crucible/evidence.py — validate_evidence_against_chain() function Severity: HIGH — prevents any experiment with dict-format affected_contracts from running

    Root cause: The function assumed affected_contracts is always a list of address strings. But the Poly Network evidence.yaml uses a dict format: 

    affected_contracts:
    EthCrossChainManager: "0x838bf9E..."
    EthCrossChainData: "0xcF2afe10..."

    The code iterated over dict keys (contract names), trying get_code("EthCrossChainManager") instead of get_code("0x838bf9E...").

    Fix applied: Added normalization logic that extracts address values from dict format before iterating.

    Bug #2: test_terminate_process_negative_pid hangs in Docker ([warn] OPEN)¶

    Location: tests/unit/test_port_lifecycle.py Severity: LOW — Docker-specific, test design issue

    Root cause: The test calls os.kill(-1, signal.SIGTERM) which sends signals to every process in the container's PID namespace. In Docker, this can hang or cause unintended side effects.

    Bug #3: BNB Token Hub — LogManager.step() message kwarg conflict (📋 KNOWN, UNFIXED)¶

    Location: attacks/bnb_token_hub_proof/bnb_token_hub_proof.py Severity: MEDIUM — blocks BNB experiment execution

    Root cause: Same issue documented as bug #9 in the original 13-bug log. The attack plugin scaffold passes message= as both positional AND keyword argument to LogManager.step().

    Bug #4: Bridge topology Polygon timeout (📋 OPEN)¶

    Location: chain_manager.py readiness timeout Severity: MEDIUM — blocks 4 experiments (Nomad, Wormhole, + hybrid variants)

    Root cause: The Polygon Anvil fork takes longer than 30 seconds to initialize when using the Ankr RPC. The readiness polling timeout is too short.
