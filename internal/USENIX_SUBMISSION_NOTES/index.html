    USENIX Submission Notes¶

    Purpose: Map Crucible's features to USENIX Security paper claims and provide AE-specific notes for evaluators.

    Paper Claim Mapping¶

    Paper Claim Crucible Feature How to Verify
    Reproducible bridge exploit reproduction Scenario engine + baseline capture/verify/restore + deterministic Anvil flags Run experiment twice, compare with crucible diff
    Evidence-backed methodology evidence.yaml + fidelity levels L0-L4 crucible plugin validate checks completeness
    Provenance tracking gather_provenance() in every run manifest run_manifest.json has git commit, foundry version, cache state
    Publication integrity Publication gates (6 rules) + artifact manifests + bundle hashes Run with --publication-mode and verify with crucible verify
    Cache hygiene CacheManager with strict/permissive policy crucible cache inspect shows provider identity, block, version
    Attack data extraction crucible attack extract auto-generates packs from tx hashes Extract a transaction, review generated files

    AE Submission Checklist¶

    • Repository tagged (e.g., v0.2.0-ae)
    • Zenodo DOI minted automatically via GitHub release
    • CITATION.cff present (needs real ORCID before tagging)
    • LICENSE file present (MIT)
    • docs/AE_WALKTHROUGH.md complete and tested
    • scripts/ae_up.sh, ae_run_demo.sh, ae_collect.sh, ae_clean.sh ready
    • Docker build verified on clean environment
    • Demo experiment verified end-to-end
    • 808 unit tests pass with no external dependencies
    • Known issues documented (see docs/internal/DEVELOPER_HANDOFF.md)

    Known Issues For Evaluators¶

    1. Nomad addresses were wrong — The scaffold originally used an EOA as the Replica contract. This has been corrected to 0xB92336759618F55bd0F8313bd843604592E27bd8 (verified on Etherscan). Ready for live fork test.

    2. 0x46e91d4f selector unresolved — The Nomad exploit transaction at 0x5ea16a4f9104b901a75662f3900ca8d3d92209d846310237ca65675e2f2e519c uses selector 0x46e91d4f, which does NOT match any standard Solidity function signature. Requires proxy ABI fetch to resolve.

    3. Multi-chain exemplar — BNB Token Hub and Horizon packs now have real research data (fork blocks, tx hashes, contract addresses, postmortems). Poly Network data also filled. All three need live Anvil fork verification.

    4. Multichain pack reclassified — The Multichain "router permit" was reclassified from L2 contract exploit to L1 MPC infrastructure narrative. No on-chain reproduction possible for this incident.

    Evaluator Instructions¶

    See docs/AE_WALKTHROUGH.md for the complete walkthrough. The key commands for evaluators:

    # Verify environment
docker compose run --rm orchestrator doctor

# Run demo experiment
docker compose run --rm orchestrator experiment run experiments/demo_experiment.yaml

# Run with publication gates
docker compose run --rm orchestrator experiment run experiments/demo_experiment.yaml --publication-mode

# Export and verify
docker compose run --rm orchestrator export <run_id>
docker compose run --rm orchestrator verify datasets/<run_id>

    Version Tagging Convention¶

    • Release tags: v1.0.0, v1.1.0, etc.
    • AE tags: v1.0.0-ae (branch from the release tag with AE-specific fixes)
    • The AE tag should be the one submitted with the paper
