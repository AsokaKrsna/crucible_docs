    Project Summary¶

    A CLI-first, modular, reproducible security research testbed for blockchain bridge vulnerability research. Built on Foundry/Anvil for EVM forking with structured JSONL telemetry and publication-quality artifact generation.

    Module Map¶

    Commands Package Structure¶

    The CLI is split into a crucible/commands/ package with clear separation of concerns:

    crucible/commands/
├── __init__.py       # Core app, CliState, shared helpers, sub-app registration
├── lifecycle.py      # init, status, doctor, fork, setup, clean
├── reset_cmd.py      # reset (soft/medium/hard), hard-reset
├── baseline.py       # baseline capture, verify
├── snapshot.py       # snapshot create, restore
├── plugin.py         # plugin list, validate
├── experiment.py     # experiment run, matrix, resume, checkpoints
├── logs.py           # logs list
├── abi.py            # abi list, validate, fetch
├── attack.py         # attack init, extract
├── cache.py          # cache inspect, fingerprint, verify, clean
├── bundle.py         # bundle hash, verify
├── export.py         # export, publish
├── analytics.py      # history, diff
├── verify.py         # verify (unified run/bundle integrity + --schemas)
├── instrument.py     # standalone instrumentation proxy
├── dashboard.py      # Streamlit dashboard launcher
└── notebook.py       # Jupyter notebook launcher

    Each sub-module registers its commands on the appropriate Typer app via decorators. Shared state (CliState) replaces the old global _DRY_RUN flag.

    Running Specific Tests¶

    # Run only unit tests (no external requirements)
pytest tests/unit/

# Run integration tests (requires Foundry)
pytest tests/integration/ -m requires_foundry

# Run with coverage
pytest tests/unit/ --cov=crucible

# Quick verification (fastest)
pytest tests/unit/test_config.py tests/unit/test_utils.py

    Architecture Changes (Current Session)¶

    Phase 1: Service Extraction¶

    • bridge_lifecycle.py — Extracted bridge deploy/configure/connect/destroy from experiment.py, eliminating the duplicate lifecycle between run() and resume()
    • provenance.py — Extracted run manifest construction into build_run_manifest(), used by _write_manifest()

    Phase 2: Scenario Step Registry¶

    • scenario_steps.py — Replaced the if/elif chain in _run_step() with a decorator-based registry dispatch. 8 step handlers registered. python_script now honors optional script: key.
    • scenario.py — Refactored to use registry, added _scenario_cache for YAML caching

    Phase 3: Manifest Normalization¶

    • plugin_manifest.py — Canonical PluginManifest dataclass with from_dict() supporting both legacy (class_name:) and modern (class:) formats, plus validate_required_fields() and validate_type()
    • All 5 legacy-format plugin manifests normalized to modern format
    • plugin.py now uses PluginManifest.from_dict() internally

    Phase 4: CLI Split¶

    • crucible/commands/ package created with 12 sub-modules
    • Global _DRY_RUN replaced with typed CliState dataclass
    • cli.py reduced to thin 3-line entry point

    Known Issues¶

    1. Nomad Replica address was wrong — The original attack plugin (attacks/nomad_replica_fraud/nomad_replica_fraud.py) used an EOA address (0x5d9d4825...) as the Replica contract. This has been corrected to 0xB92336759618F55bd0F8313bd843604592E27bd8 (verified on Etherscan). The attack plugin was updated accordingly.

    2. 0x46e91d4f selector unresolved — The exploit transaction at 0x5ea16a4f9104b901a75662f3900ca8d3d92209d846310237ca65675e2f2e519c uses selector 0x46e91d4f, which does NOT match any standard Solidity function signature (process(bytes) = 0x61bd0e46, process(bytes,bytes) = 0x7c39d130). It likely targets a private/internal function on the UpgradeBeaconProxy implementation. Requires proxy ABI fetch to resolve.

    3. Keccak256 implementation — Python's hashlib.sha3_256() is NIST SHA-3 (FIPS 202), NOT Ethereum's keccak256. Always use Web3.keccak(bytes) or Crypto.Hash.keccak.new(digest_bits=256) for Ethereum-compatible hashing. This bug was caught in code review multiple times.

    4. ABI format variance — Sourcify returns {"abi": [...]} while Etherscan returns the raw ABI array as a string. Normalization is handled in AbiRegistry._fetch_* methods.

    Adding A New Attack¶

    For a complete step-by-step guide, see docs/ATTACK_PACK_GUIDE.md.

    Quick Path (from tx hash — 10 seconds)¶

    crucible attack extract <tx_hash> --name my_attack --chain ethereum

    This generates a complete plugin scaffold. Customize: - attacks/my_attack/my_attack.py — add real exploit logic - attacks/my_attack/evidence.yaml — fix fidelity level, add postmortems - scenarios/my_attack_pre_exploit/scenario.yaml — adjust fork block

    Manual Path¶

    mkdir -p attacks/my_attack
# Create: plugin.yaml, my_attack.py, evidence.yaml
mkdir -p scenarios/my_attack_pre_exploit
# Create: scenario.yaml
crucible plugin validate attacks/my_attack
