    Running LLM Agents in Crucible¶

    This guide explains how to configure and run LLM-driven autonomous agents in the Crucible testbed. Agents replace traditional hand-coded attack plugins with an AI loop: the agent observes the forked chain environment, decides which tools to call, executes them, and repeats until done.

    Quick Start¶

    # 1. Ensure your .env has the API key for your chosen provider
#    (Ollama users: no key needed — just have Ollama running locally)

# 2. Run the demo agent experiment (simple chain exploration)
crucible experiment run experiments/demo_agent.yaml

# 3. Or run the Nomad exploit agent (targets a real bridge vulnerability)
crucible experiment run experiments/llm_exploit_agent.yaml

    How It Works¶

    The agent lifecycle has five stages:

    1. Prepare — The agent loads its configuration from the experiment YAML's run: block, including the LLM provider, model, privilege level, task, and objective. Provider is auto-configured via the from_run_config() factory (see below).
    2. Observe — The agent gathers the current state: tool history, available tools, and recent results.
    3. Think — The agent sends the current state + conversation history to the LLM provider. The LLM decides which tool to call next (or signals completion).
    4. Act — The agent executes the chosen tool via the Crucible RPC layer (impersonation, balance checks, storage reads, etc.).
    5. Repeat — Steps 2–4 loop until is_done() returns True or the agent signals completion.
    6. Verify — The agent verifies the outcome (by default, always passes).

    Experiment YAML Configuration¶

    Create an experiment YAML file with the following structure:

    name: my_agent_experiment
scenario: demo_scenario          # any scenario with a single-chain topology
attack: demo_agent               # plugin name from attacks/<name>/plugin.yaml
topology: single

run:
  # ── Provider Configuration ────────────────────────────────────
  agent_provider: openai         # openai | anthropic | ollama | local | custom
  agent_model: gpt-4o            # model name for the chosen provider
  # agent_base_url: ...          # optional: override the provider's default base URL

  # ── Agent Runtime Configuration ───────────────────────────────
  agent_privilege: medium        # low | medium | high
  agent_temperature: 0.3         # Lower = more deterministic (default: 0.7)
  agent_max_iterations: 30       # Max tool calls before timeout (default: 50)
  agent_llm_budget: 10000        # Max total tokens (stops agent if exceeded)
  agent_task: "Explore the forked chain and report findings"
  agent_objective: "Use available tools to check chain ID, balances, and contract code"
  agent_allowed_accounts: []     # empty = no address restriction

  # ── Recording for Deterministic Replay ──────────────────────
  # record_path: tests/fixtures/my_agent.jsonl  # uncomment to capture RPC fixture

  # ── Standard Experiment Config ────────────────────────────────
  traces: false
  storage_diffs: true
  gas_profiling: true

tags:
  - agent
  - demo

    Provider Types¶

    Type Default Model API Key Required Notes
    openai gpt-4o OPENAI_API_KEY in .env Also works with Azure, vLLM, etc. via agent_base_url
    anthropic claude-sonnet-4-20250514 ANTHROPIC_API_KEY in .env Uses Anthropic Messages API
    ollama llama3.1 No Auto-connects to http://localhost:11434/v1
    local local-model Optional Requires explicit agent_base_url
    custom local-model Optional Requires explicit agent_base_url

    Privilege Levels¶

    Level Tools Available Description
    low 6 tools (read-only) chain_id, get_balance, get_code, get_storage, call_contract, get_logs
    medium 8 tools Low tools + impersonate, send_transaction
    high 9 tools Medium tools + advance_time

    Advanced Configuration¶

    Key Default Description
    agent_temperature 0.7 LLM sampling temperature. Lower values (0.1–0.3) produce more deterministic, focused behaviour. Higher values encourage creativity.
    agent_max_iterations 50 Maximum number of tool calls before the agent loop stops. Prevents runaway agents.
    agent_llm_budget None (unlimited) Maximum total tokens the LLM provider can consume. When the budget is exceeded, the agent stops and logs a budget_exceeded warning. A 90% threshold warning is also logged. Set to a reasonable value like 10000 to cap costs.
    agent_allowed_accounts [] (all) List of allowed Ethereum addresses the agent can interact with. Empty = no restriction.
    record_path None Path to a JSONL file where all RPC calls are recorded for later replay. See RPC Recording.

    Provider Auto-Configuration (from_run_config)¶

    When you run crucible experiment run, the experiment runner automatically detects agent_provider in the experiment YAML's run: block and creates the correct LLM provider via the from_run_config() factory on LLMAgentPlugin.

    This means you don't need to modify Python code to change providers — just edit the YAML:

    # Switch from Ollama to OpenAI:
agent_provider: openai
agent_model: gpt-4o

# Or use Anthropic:
agent_provider: anthropic
agent_model: claude-sonnet-4-20250514

# Or a custom OpenAI-compatible endpoint:
agent_provider: custom
agent_model: my-model
agent_base_url: http://my-proxy:8080/v1

    The prepare() method in LLMAgentPlugin checks the _configured_from_run_config flag and skips redundant provider re-creation when the factory was used.

    Included Agent Packs¶

    Crucible ships with two agent packs:

    attacks/demo_agent/ — Simple Chain Explorer¶

    A minimal example that explores a forked chain, checks chain ID, balances, and contract code. Best starting point for understanding the agent system.

    crucible experiment run experiments/demo_agent.yaml

    attacks/llm_exploit_agent/ — Nomad Bridge Exploit Agent¶

    A more sophisticated agent targeting the Nomad bridge Replica contract (0xB923...). The agent:

    1. Verifies the Replica contract exists and has code
    2. Reads the current committedRoot (storage slot 0x1)
    3. Constructs a forged process(bytes,bytes) call using the function selector 0x61bd0e46
    4. Sends the transaction to exploit the authentication bypass
    5. Verifies the committedRoot changed
    crucible experiment run experiments/llm_exploit_agent.yaml

    Requires: A running LLM provider (Ollama by default) and an archive RPC URL for Ethereum at fork block 15,259,100 (the Nomad pre-exploit block).

    Creating a Custom Agent Plugin¶

    To create a custom agent, subclass LLMAgentPlugin and override the methods you need:

    from crucible.agent_loop import LLMAgentPlugin
from crucible.agent_tools import AgentContext

class MyAgent(LLMAgentPlugin):
    def metadata(self) -> dict:
        return {
            "name": "my_agent",
            "version": "1.0.0",
            "type": "agent",
        }

    def build_objective(self, agent_ctx: AgentContext) -> str:
        """Custom objective for your agent."""
        return "Your custom objective here."

    Then register it in attacks/<name>/plugin.yaml:

    name: my_agent
type: agent
version: "1.0.0"
entry: my_agent.py
class: MyAgent
requires:
  crucible: ">=1.0.0"
  chains:
    - ethereum
  topology: single

    Available Override Points¶

    Method Purpose Called When
    metadata() Return plugin metadata (name, version, type) Plugin loading
    build_objective(ctx) Return the objective string for the LLM Each think() cycle
    build_system_prompt(ctx) Return the full system prompt (override for full control) First think() cycle
    prepare(context, agent_ctx) Set up agent state before the main loop Before the agent loop starts
    observe(context, agent_ctx) Gather observations from the environment Each loop iteration
    think(observation, context, agent_ctx) Decide the next action given the observation Each loop iteration
    respond(result, context, agent_ctx) Process the result of a tool call After each tool execution
    is_done(context, agent_ctx) Check if the agent should stop Each loop iteration
    verify(context, agent_ctx) Verify the agent's outcome After the loop ends

    Reasoning Chain Logging¶

    Every agent run produces an agent_reasoning.jsonl file in the run's log directory. Each line is a JSON record containing:

    {
  "step": 5,
  "observation": "## Current State\n  Privilege: medium\n  Tools available: 8\n  Total tool calls so far: 4\n\n## Most Recent Tool Results\n\n  \u2713 chain_id:\n    {\"chain_id\": 1337}\n\n  \u2713 get_balance:\n    {\"balance\": \"1000000000000000000\"}\n",
  "action": {
    "tool": "get_code",
    "params": {"address": "0xB92336759618F55bd0F8313bd843604592E27bd8"}
  },
  "conversation_snapshot": [
    {"role": "system", "content": "You are an autonomous security research agent..."},
    {"role": "user", "content": "## Current State..."},
    {"role": "assistant", "content": "I will check the contract code at..."}
  ]
}

    This provides a full reasoning trace for reproducibility and review.

    RPC Recording for Deterministic Replay¶

    Agent runs can be recorded to a fixture file for deterministic replay:

    # Run the agent experiment with recording enabled
crucible experiment run experiments/llm_exploit_agent.yaml \
  --record tests/fixtures/llm_exploit_agent.jsonl

    Or set record_path in the experiment YAML's run: block:

    run:
  record_path: tests/fixtures/llm_exploit_agent.jsonl

    The recorded fixture can then be served by the ReplayServer (from crucible.recording) for hermetic tests that don't need network access. See the existing tests/e2e/test_hermetic_nomad.py for a complete example.

    Budget and Token Management¶

    The agent_llm_budget config caps total token consumption:

    run:
  agent_llm_budget: 5000        # Hard stop at 5000 total tokens

    When the budget is configured:

    • The agent checks the budget before each LLM call and stops if exceeded
    • A warning is logged at 90% of the budget
    • The budget status is reported in verify() output
    • The agent loop's is_done() also checks the budget every iteration

    Budget takes precedence from the constructor if both constructor and run_config set it.

    Offline / Testing Without an LLM¶

    To test the agent lifecycle without an actual LLM call, create a subclass that overrides think():

    class TestAgent(LLMAgentPlugin):
    def think(self, observation, context, agent_ctx):
        """Return a fixed action for testing."""
        from crucible.agent_tools import AgentAction
        return AgentAction(tool="chain_id", params={})

    You can also use a mock provider (see tests/unit/test_llm_providers.py for examples):

    from unittest.mock import MagicMock
from crucible.llm_providers import LLMProvider, LLMResponse

mock = MagicMock(spec=LLMProvider)
mock.model_name = "mock-model"
mock.chat.return_value = LLMResponse(
    content='{"status": "done", "summary": "test"}'
)

agent = LLMAgentPlugin(provider=mock)

    Troubleshooting¶

    Symptom Likely Cause Fix
    OpenAI API key not set Missing OPENAI_API_KEY in .env Add the key or switch provider to ollama
    Anthropic API key not set Missing ANTHROPIC_API_KEY in .env Add the key
    Agent plugin 'xyz' not found Plugin YAML has wrong type: or path Ensure type: agent in plugin.yaml
    Agent loops without making progress Model too small or objective too vague Use a capable model (gpt-4o, claude-sonnet-4) and a specific objective. Lower agent_temperature to 0.1–0.3 for more focused behaviour.
    Agent stops immediately with budget_exceeded Budget is too low for the objective Increase agent_llm_budget or disable it (remove the key)
    can't connect to localhost:11434 Ollama not running or reachable Start Ollama (ollama serve) and pull the model (ollama pull llama3.1). For a remote Ollama instance, set agent_base_url: http://<remote-ip>:11434/v1 in the experiment YAML.
    Ollama returns empty tool calls Model doesn't support function calling Use a model with tool-calling support (llama3.1:70b, mistral:7b, qwen2.5:7b)
    llama-server process has terminated: signal: segmentation fault CPU-only compatibility issue with some Intel Xeon/older processors Update Ollama to the latest version (curl -fsSL https://ollama.com/install.sh | sh), or try qwen2:0.5b (352MB, lower CPU requirements). If it still segfaults, use a remote Ollama instance or switch to an OpenAI/Anthropic provider with an API key.
    Ollama segfaults on model load (CPU-only) Specific CPU architectures may have llama.cpp compatibility issues Workarounds: (1) Run Ollama on a different machine and set agent_base_url to its URL. (2) Add num_threads: 2 to the generate options. (3) Switch to a cloud API provider (OpenAI, Anthropic — add API key to .env).
    experiment run doesn't load agent config agent_provider not set in YAML Add agent_provider: ollama to the run: block
    Agent ignores the objective Prompt engineering issue Make the objective concrete and step-by-step. See attacks/llm_exploit_agent/llm_exploit_agent.py for a detailed example.
