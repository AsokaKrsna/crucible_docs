    Deployment Guide¶

    How to deploy Crucible on a target system via SSH and Docker. Last updated: June 2026

    Table of Contents¶

    1. Prerequisites
    2. SSH Clone
    3. Docker Setup
    4. Running with Docker Compose
    5. Running Unit Tests in Docker
    6. Environment Configuration
    7. Production Deployment

    1. Prerequisites¶

    Target System Requirements¶

    Requirement Minimum Recommended
    OS Linux (Ubuntu 22.04+) Ubuntu 24.04 LTS
    CPU 2 cores 4+ cores
    RAM 8 GB 16 GB
    Disk 20 GB free 100 GB+ SSD
    Python 3.11 3.12
    Docker 24.0+ 27.0+
    Docker Compose v2.24+ v2.29+

    Required Tools¶

    # Install Docker (Ubuntu/Debian)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
# Log out and back in, or run: newgrp docker

# Verify installation
docker --version
docker compose version

    Required Access¶

    • SSH key configured on the target system for passwordless login
    • Git access to the repository (public or private)
    • Archive RPC URL for chain forking (see RPC Provider Guide)

    2. SSH Clone¶

    Clone the repository to the target system via SSH:

    # One-time setup: add the target system's SSH key to your GitHub/GitLab account

# Clone with SSH
git clone git@github.com:crucible/testbed.git crucible
# Or for a private repo:
# git clone git@github.com:your-org/crucible.git crucible

cd crucible

# Verify the clone
git log --oneline -5
git status

    Using a Deployment Key¶

    For automated deployments, create a dedicated SSH deploy key:

    # On the target system, generate a key (no passphrase for automation)
ssh-keygen -t ed25519 -C "crucible-deploy" -f ~/.ssh/crucible_deploy_key

# Add the public key (~/.ssh/crucible_deploy_key.pub) to your Git provider:
# - GitHub: Repository → Settings → Deploy Keys → Add deploy key
# - GitLab: Repository → Settings → Repository → Deploy Keys

# Configure SSH to use the deploy key
cat >> ~/.ssh/config << 'EOF'
Host github.com
    HostName github.com
    IdentityFile ~/.ssh/crucible_deploy_key
    IdentitiesOnly yes
EOF

    Keeping the Clone Updated¶

    # Pull latest changes
git pull origin master

# Or use a deployment script (scripts/ae_up.sh)
./scripts/ae_up.sh

    3. Docker Setup¶

    Option A: Quick Start with Docker Compose¶

    cd crucible

# 1. Configure environment
cp .env.example .env
# Edit .env with your RPC URLs:
#   ETH_RPC_URL=https://eth.llamarpc.com

# 2. Build the Docker images
docker compose build orchestrator
docker compose build dashboard

# 3. Verify the build
docker compose run --rm orchestrator doctor

    Option B: Build and Run Script¶

    Use the pre-built scripts for a complete setup:

    # Build and run a full end-to-end demo
./scripts/ae_up.sh

    This script will: 1. Build the Docker images 2. Run crucible doctor to validate the environment 3. Run unit tests inside the container 4. Start the dashboard and chain services

    .dockerignore¶

    The project includes a .dockerignore file that excludes: - .venv/ — prevents host Python environment from overriding the container's - states/, logs/, datasets/, .fork_cache/, cache/, out/ — runtime generated files - .git/, .gitignore, .gitmodules — version control - __pycache__/, *.pyc, .pytest_cache/ — Python cache files - .env — secrets and credentials - *.md — documentation (not needed at runtime)

    4. Running with Docker Compose¶

    Basic Commands¶

    # Run crucible commands
docker compose run --rm orchestrator doctor
docker compose run --rm orchestrator status
docker compose run --rm orchestrator init
docker compose run --rm orchestrator plugin list
docker compose run --rm orchestrator experiment run experiments/demo_experiment.yaml --no-baseline

# Start the dashboard
docker compose up -d dashboard
# Open http://localhost:8501 in your browser

# Start Anvil chains (requires .env with RPC URLs)
docker compose up -d anvil-ethereum
docker compose up -d anvil-polygon

# View logs
docker compose logs -f orchestrator

# Stop all services
docker compose down

    Running with Full Chain Support¶

    # docker-compose.override.yml (local file, gitignored)
# Start the chains you need:
docker compose up -d anvil-ethereum
docker compose up -d anvil-polygon

# Then run experiments:
docker compose run --rm orchestrator experiment run experiments/nomad_replay.yaml

    The Entrypoint Script¶

    When you run any docker compose run --rm orchestrator <command>, the container:

    1. Runs uv sync — syncs the frozen environment (all extras: dev, ui, analysis)
    2. This ensures the container has all dependencies even if the host's .venv/ is mounted
    3. Takes ~3-5 seconds on first run, then cached
    4. Executes the command via uv run crucible <command>

    This means docker compose run --rm orchestrator doctor is equivalent to running crucible doctor on a properly set up local environment.

    5. Running Unit Tests in Docker¶

    # Run all unit tests (520+ tests)
docker compose run --rm --entrypoint bash orchestrator -c "\
  cd /workspace && \
  uv run pytest tests/unit/ -q --tb=short"

# Run specific test file
docker compose run --rm --entrypoint bash orchestrator -c "\
  cd /workspace && \
  uv run pytest tests/unit/test_config.py -v"

# Run with coverage
docker compose run --rm --entrypoint bash orchestrator -c "\
  cd /workspace && \
  uv run pytest tests/unit/ --cov=crucible --cov-report=term"

    Note: Dashboard e2e tests (test_dashboard_e2e.py) are excluded from the Docker smoke test because they require the Streamlit testing framework's runtime environment. These pass when run locally.

    6. Environment Configuration¶

    Required Environment Variables¶

    Create .env from the template:

    cp .env.example .env

    At minimum, set ETH_RPC_URL to an archive-node RPC provider:

    # Required for chain forking
ETH_RPC_URL=https://eth.llamarpc.com

    Optional Variables¶

    # Multi-chain support
POLYGON_RPC_URL=https://polygon.llamarpc.com
BSC_RPC_URL=https://bsc.publicnode.com
HARMONY_RPC_URL=https://harmony.publicnode.com

# Paid providers (higher rate limits)
ALCHEMY_KEY=your_key_here

# Etherscan API key (for ABI fetching)
ETHERSCAN_API_KEY=your_key_here

# LLM API keys (for autonomous agent experiments)
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

    Provider Notes¶

    Provider Free Tier Rate Limit Archive Support Best For
    LlamaRPC 25 req/s [ok] Yes Development, single-chain
    Ankr 15 req/s [ok] Yes Fallback provider
    PublicNode 10 req/s [ok] Yes Last resort fallback
    Alchemy 30 req/s (free tier) [ok] Yes Production, multi-chain
    Infura 25 req/s (free tier) [ok] Yes Production, multi-chain

    7. Production Deployment¶

    Docker Compose with Override¶

    For production environments, create docker-compose.override.yml (gitignored):

    version: "3.8"
services:
  dashboard:
    ports:
      - "8501:8501"
    restart: always
    environment:
      - STREAMLIT_SERVER_RUNONSAVE=false

  anvil-ethereum:
    restart: always
    ports:
      - "8545:8545"

  anvil-polygon:
    restart: always
    ports:
      - "8546:8546"

    Running as a Service¶

    Create a systemd service file (/etc/systemd/system/crucible.service):

    [Unit]
Description=Crucible Bridge Security Research Testbed
After=docker.service
Requires=docker.service

[Service]
WorkingDirectory=/opt/crucible
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
Restart=always
User=root
Group=docker

[Install]
WantedBy=multi-user.target

    Then:

    sudo systemctl daemon-reload
sudo systemctl enable crucible
sudo systemctl start crucible
sudo systemctl status crucible

    Resource Limits¶

    Configure Docker resource limits in /etc/docker/daemon.json:

    {
  "default-shm-size": "256m",
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

    Monitoring¶

    # Check container health
docker compose ps

# View resource usage
docker stats

# Check disk usage
docker compose run --rm orchestrator status --disk

# View logs
docker compose logs -f --tail=100

    8. Troubleshooting¶

    Common Issues¶

    Symptom Cause Fix
    ModuleNotFoundError: No module named 'streamlit' Host .venv/ mounted over container's The entrypoint script handles this automatically. If still failing, run docker compose build --no-cache orchestrator
    pytest: command not found Missing dev dependencies The entrypoint installs --extra dev. If still failing, run docker compose build --no-cache orchestrator
    anvil: command not found Foundry not installed in Docker Run docker compose build --no-cache orchestrator to re-install Foundry
    Error: No such service: anvil-ethereum Anvil services not started Run docker compose up -d anvil-ethereum first
    Connection refused on RPC Anvil not ready yet Wait 30-60s for Anvil to finish forking. Check with docker compose logs anvil-ethereum
    Port 8501 already in use Another Streamlit instance running Change port: docker compose run --rm -p 8502:8501 dashboard
    Build fails with uv lock --frozen error Lockfile out of sync Run uv lock locally, commit the updated uv.lock, then rebuild

    Checking Container Health¶

    # Check if container is running
docker compose ps

# Check logs
docker compose logs orchestrator --tail=50

# Run a quick health check
docker compose run --rm orchestrator doctor

# Verify Python environment
docker compose run --rm --entrypoint bash orchestrator -c "\
  uv run python -c \"import streamlit; print('streamlit OK')\" && \
  uv run python -c \"import pandas; print('pandas OK')\" && \
  uv run python -c \"import crucible; print('crucible OK')\""

    Resetting the Environment¶

    # Stop everything
docker compose down -v  # -v removes volumes (states, logs, cache)

# Rebuild from scratch
docker compose build --no-cache orchestrator

# Start fresh
docker compose run --rm orchestrator init
