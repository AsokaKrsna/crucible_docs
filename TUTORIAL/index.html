    Tutorial: Reproduce the Nomad Bridge Hack in 10 Minutes¶

    What you'll accomplish: Fork Ethereum at block 15,259,100, run the Nomad Bridge Replica exploit, verify the forged transaction was accepted, confirm the reproduction is deterministic across two runs, and export a verifiable publication bundle.

    Total time: ~10 minutes (2 min setup + 5 min first run + 3 min second run) Dependencies: Docker (or Python 3.11+ + Foundry)

    Before You Start¶

    Make sure you have an Ethereum archive RPC endpoint. If you don't have one, create a free account at LlamaNodes — their free tier works for all Crucible experiments.

    1. Clone the repo and configure RPC
    git clone <repo-url> && cd crucible
cp .env.example .env
    1. Edit .env and set your archive RPC URL:
    ETH_RPC_URL=https://eth.llamarpc.com   # or your provider
    1. Build the Docker image (one-time, ~3 min):

    bash docker compose build> Local alternative: If you have Python 3.11+ and Foundry installed,

    run uv sync && source .venv/bin/activate && crucible init instead.

    Docker Caveats¶

    Fork block management: Different experiments need different fork blocks. The shared testbed.yaml has one block per chain. Override per experiment using testbed.local.yaml (gitignored, deep-merged):

    # testbed.local.yaml — overrides testbed.yaml chains section
chains:
  ethereum:
    fork_block: 15259100  # Nomad Bridge fork block

    Stale Anvil meta files: When switching topologies or re-forking, old .meta.json files in the states/ Docker volume cause connection errors. Always clean before forking:

    rm -f states/*.meta.json

    Chaining commands requires --entrypoint bash: The orchestrator's entrypoint is uv run crucible, not a shell. To run multiple commands (fork + experiment), use:

    docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"

    Streamlit not available in orchestrator: The crucible doctor check for streamlit will fail — this is expected. Streamlit runs in a separate dashboard service (docker compose up -d dashboard), not in the orchestrator.

    .venv/CACHEDIR.TAG warning: A benign error about removing /workspace/.venv/CACHEDIR.TAG may appear on startup. It occurs because the host's .venv is mounted into the container. Safe to ignore.

    Step-by-Step Commands¶

    Step 1: Validate your environment¶

    docker compose run --rm orchestrator doctor

    Expected output (all checks pass):

    Checks:
  [ok] Python imports — all present
  [ok] Foundry — forge 1.0.0 (stable)
  [ok] Config — testbed.yaml loaded
  [ok] RPC — ETH_RPC_URL resolves
  [ok] Directories — workspace, logs, datasets, states
  [ok] Docker — running

    If any check fails, fix the issue before continuing. The most common failure is a missing or incorrect ETH_RPC_URL in your .env file.

    Tip: The Nomad experiment declares topology: bridge but this tutorial uses --topology single. If the runtime fidelity check fails with a "trace artifacts" error due to this mismatch, add --no-fidelity to skip the L2 verification: uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline --no-fidelity

    Step 2: Fork Ethereum and run the exploit¶

    This is the core command — it forks the chain and runs the experiment in one container. The --rm flag destroys the container after it exits, so both operations must be in the same --entrypoint bash chain.

    docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"

    Expected output:

    [ok] Starting chain 'ethereum' (fork block 15259100)...
   Forking Ethereum at block 15,259,100...
[ok] Chain 'ethereum' ready on http://127.0.0.1:8545

    Step 5: Run a second time (prove determinism)¶

    docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json && \
  uv run crucible fork --topology single && \
  uv run crucible experiment run experiments/nomad_replay.yaml --no-baseline
"

    Expected output: Same as Step 2 — identical artifact stream, but with a different run ID. Note it down as RUN_B (e.g., DS-2026-00002):

    📡 Completed: DS-2026-00002   (experiment run time: 4.9s)

    The gas (25,150), storage diffs, events, and balance changes should be identical to the first run. Let's prove it.

    Step 6: Diff the two runs¶

    Replace DS-2026-00001 and DS-2026-00002 with your actual run IDs:

    docker compose run --rm orchestrator diff DS-2026-00001 DS-2026-00002

    Expected output:

    {
  "storage_equal": true,
  "events_equal": true,
  "gas_equal": true,
  "balances_equal": true,
  "gas": {"run_a": 25150, "run_b": 25150, "delta": 0},
  "storage": {"run_a": 2, "run_b": 2},
  "events": {"run_a": 2, "run_b": 2},
  "balances": {"run_a": 2, "run_b": 2}
}

    All four must be true for a deterministic reproduction. If any are false, the reproduction is non-deterministic (file a bug).

    Replace DS-2026-00001 with your first run ID:

    Step 8: Verify the bundle integrity¶

    docker compose run --rm orchestrator verify datasets/DS-2026-00001

    Expected output:

    [ok] Verification: PASS (2 checks)
   [ok]  Run integrity    (log_integrity.json)
        4 file(s) verified
   [ok]  Bundle integrity (artifact_bundle_manifest.json)
        12 file(s) verified
All artifacts intact at: ...datasets/DS-2026-00001

    The crucible verify command runs two checks automatically: 1. Run integrity — re-computes SHA-256 hashes against the original log_integrity.json 2. Bundle integrity — re-computes SHA-256 hashes against artifact_bundle_manifest.json

    To see which specific files were hashed, add the --verbose flag:

    docker compose run --rm orchestrator verify datasets/DS-2026-00001 --verbose

    This proves the exported artifacts haven't been tampered with. Anyone who runs crucible verify on this directory will get the same result — any hash mismatch, missing file, or extra file is reported immediately.

    What Just Happened¶

    The Vulnerability¶

    The Nomad bridge's Replica contract at 0xB92336759618F55bd0F8313bd843604592E27bd8 was deployed with its committedRoot (storage slot 1) initialized to 0x00. The process(bytes message, bytes proof) function verifies message authenticity by checking a Merkle proof against this root:

    function process(bytes memory _message, bytes memory _proof) external returns (bool) {
    bytes32 _messageHash = keccak256(abi.encodePacked(_message));
    require(messages[_messageHash] == MessageStatus.NONE, "Replica: message already processed");
    require(MerkleProof.verify(_proof, committedRoot, _messageHash), "Replica: proof verification failed");
    // ... execute the message
}

    Since a Merkle proof with an all-zeros root still passes verification (a hash-chain of 32 zero bytes remains 0x00), any message can be forged.

    What Crucible Did¶

    Phase What Happened
    Chain fork Crucible started Anvil with --fork-block-number 15259100, creating a local replica of Ethereum at block 15,259,100 — the block before the first exploit transaction.
    Proxy attach Crucible started a transparent RPC proxy between the attack plugin and Anvil. Every RPC call is intercepted.
    Precondition check The attack plugin read storage slot 1 of the Replica contract and confirmed committedRoot == 0x00.
    Exploit execution The plugin forged a minimal cross-chain message (32 zero bytes for the root + empty fields), encoded it as process(bytes,bytes) calldata, and sent it as a real eth_sendTransaction.
    Auto-instrumentation The RPC proxy automatically captured: the transaction receipt (gas), contract events (MessageAccepted), storage diffs (messages[hash] → ACCEPTED), and balance changes (attacker paid gas).
    Verification The plugin re-read storage to confirm the message status changed from 0 (NONE) to 1 (PROCESSED).

    What Makes This Reproducible¶

    1. Fork block pinned — Every run forks the same block, ensuring identical starting state
    2. Deterministic Anvil flags — gas_price: 0, order: fifo, timestamp: fork_block — all transactions produce identical results
    3. SHA-256 integrity — Every artifact file is hashed, making tampering detectable
    4. Provenance recording — run_manifest.json captures git commit, Foundry version, Python version, config fingerprint — the full environment identity

    The Artifact Streams¶

    Stream Records What to Look For
    events.jsonl All emitted events MessageAccepted with root 0x00
    storage_diffs.jsonl Every slot that changed messages[hash]: 0x00 → 0x01
    gas_profile.jsonl Gas per transaction Exactly 25,150 gas for the exploit
    balance_diffs.jsonl ETH balance changes Attacker paid gas
    run.jsonl Lifecycle events 18 structured log entries

    Cleanup¶

    # Stop any running chains and remove temp files
docker compose run --rm orchestrator clean --all

# Remove Docker volumes (deletes all logs and datasets)
docker compose down -v
