    RPC Provider Guide¶

    Why Archive Nodes Are Required¶

    Crucible reproduces historical blockchain exploits by forking the chain at a specific block using Foundry's anvil. To do this, anvil needs an archive node (also called a "full archive node") that can serve historical state data.

    Archive vs Full Node¶

    Feature Full Node Archive Node
    Latest block state [ok] [ok]
    Historical state (eth_getBalance at old block) [fail] (pruned after ~128 blocks) [ok] (stores ALL state since genesis)
    eth_getBlockByNumber (old blocks) [ok] (block headers kept) [ok]
    eth_getCode / eth_call (old blocks) [fail] [ok]
    Disk space (Ethereum) ~1 TB ~12+ TB
    Disk space (BSC) ~1 TB ~10+ TB

    Which Chains Need Archive Access¶

    Attack Chains Forked Fork Block Archive Required?
    Nomad Bridge Ethereum + Polygon 15,259,100 / — [ok] Ethereum [ok] Polygon
    Wormhole Guardian Ethereum 14,125,553 [ok] Ethereum
    Poly Network Ethereum (only) 12,996,693 [ok] Ethereum
    Poly Network (full multi-chain) Ethereum + BSC + Polygon 12,996,693 / 9,755,172 / 16,332,982 [ok] All 3 chains

    Key insight: All three attacks need Ethereum archive. Ethereum has the best free archive support. For multi-chain (BSC + Polygon at 2021-era blocks), paid archive providers are required.

    Configuration in Crucible¶

    Step 1: Get an API Key¶

    1. Sign up at GetBlock
    2. Go to Dashboard → Create Access Token
    3. Select BSC Mainnet and Polygon Mainnet
    4. Copy the provided endpoint URLs

    Step 2: Set Environment Variables¶

    Add to your .env file:

    # Ethereum archive — ANKR free tier (sign up at https://app.ankr.com)
# LlamaNodes (eth.llamarpc.com) also works but may rate-limit Anvil
ETH_RPC_URL=https://rpc.ankr.com/eth/YOUR_ANKR_API_KEY

# BSC Archive (GetBlock or your provider)
BSC_RPC_URL=https://go.getblock.io/YOUR_ACCESS_TOKEN

# Polygon Archive (GetBlock or your provider)
POLYGON_RPC_URL=https://go.getblock.io/YOUR_ACCESS_TOKEN

    Step 3: Configure Rate Limiting¶

    Crucible uses an internal TokenBucket algorithm to ensure Anvil does not exceed your provider's rate limits (which would cause 429 Too Many Requests errors and experiment failures).

    You can configure rate limits per-provider in testbed.yaml:

    rpc_providers:
  ethereum:
    - name: ankr_free
      url: "${ETH_RPC_URL}"
      rate_limit: 30       # 30 requests per second
      burst_limit: 50      # Allow temporary bursts up to 50

    If Anvil requests data faster than the rate_limit, the proxy will pause the request until tokens replenish.

    Step 4: Verify Connectivity¶

    # Quick test from orchestrator container
docker compose run --rm --entrypoint bash orchestrator -c '
curl -s -X POST -H "Content-Type: application/json" \
  -d "{\"jsonrpc\":\"2.0\",\"id\":1,\"method\":\"eth_blockNumber\",\"params\":[]}" \
  https://go.getblock.io/YOUR_ACCESS_TOKEN
'

    Step 4: Run Multi-Chain Experiment¶

    # Update fork blocks in testbed.yaml first (or use testbed.local.yaml)
docker compose run --rm --entrypoint bash orchestrator -c "
  rm -f states/*.meta.json
  uv run crucible fork --topology poly_network
  uv run crucible experiment run experiments/poly_network_replay.yaml --no-baseline
"

    Troubleshooting¶

    "Historical state not available" Error¶

    Error: Failed to get block for block number: 9755172

    Cause: The RPC endpoint is a full node, not an archive node. It doesn't store historical state.

    Fix: Switch to an archive-capable provider (GetBlock, QuickNode, etc.).

    "anvil_setBalance" Fails with "Historical State Not Available"¶

    RPC error for anvil_setBalance: failed to get account for 0xDeaDDeaD...:
server returned an error response: error code -32000:
state 0x34f7657d... is not available

    Cause: When anvil_setBalance is called on an address that doesn't exist in the forked state, Anvil tries to look up the address in the remote archive. If the archive node doesn't have full historical state at the fork block, it returns "state not available". This is most common on Polygon and BSC archives at older blocks (2021-2022).

    Why it happens: 1. Crucible starts Anvil with --accounts 0 --balance 0 (no prefunded test accounts) 2. The bridge plugin calls anvil_setBalance(deployer, amount) to fund a deployer address 3. Since the deployer address doesn't exist in the forked state, Anvil queries the remote archive 4. If the archive has pruned historical state (common with free Polygon/BSC providers), the query fails

    Note: This also affects eth_sendTransaction — even with gasPrice=0, Anvil checks if the sender account exists in the forked state, which triggers the same remote lookup.

    Fix (already applied to all 3 bridge plugins): Per-chain deployment is wrapped in try/except in bridges/nomad_bridge/nomad_bridge.py, bridges/poly_network_bridge/poly_network_bridge.py, and bridges/wormhole_bridge/wormhole_bridge.py. If a chain's archive can't serve state, it's skipped gracefully with a warning.

    Workaround: - Use a different RPC provider with better archive depth (GetBlock typically works for all 3 chains) - Test archive depth with: 

    # This queries an address that definitely exists at the fork block
curl -s -X POST -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"eth_getBalance","params":["0x838bf9E95CB12Dd76a54C9f9D2E3082EAF928270", "0xE7E30C"]}' \
  https://your-rpc-provider
# 0xE7E30C = 15197964 (Poly Network fork block on Ethereum)
# Returns "0x..." if archive works, error if not
    - If a chain's archive is unreliable, run experiments with only the chains that work (use a topology without the problematic chain)

    "eth_sendTransaction" From New Address Fails¶

    RPC error for eth_sendTransaction: failed to get account for 0xDeaDDeaD...:
historical state ... is not available

    Cause: Same root cause as the anvil_setBalance error above. Even sending a transaction with gasPrice=0 from a new (non-existent) address triggers a remote archive lookup.

    Fix: Same as above — the bridge plugins handle this gracefully now. For custom code, use an address that already exists in the forked state (like a real contract address) instead of a new random EOA.

    How To Test Archive Depth¶

    Run this from the orchestrator container to verify a provider has the required historical state:

    docker compose run --rm --entrypoint bash orchestrator -c '
  # Use the hex block number for the chain you are testing:
  #   Ethereum (Nomad):   0xE8DB9C  (15,259,100)
  #   Ethereum (PolyNet): 0xC654B5  (12,996,693)
  #   Polygon:            0xF91D96  (16,332,982)
  #   BSC:                0x94E334  (9,755,172)
  HEX_BLOCK=0xE8DB9C
  RPC_URL="$ETH_RPC_URL"

  # Test 1: Can the provider serve a block by number?
  echo "=== Test 1: Block exists ==="
  curl -s --max-time 10 -X POST -H "Content-Type: application/json" \
    -d "{\"jsonrpc\":\"2.0\",\"id\":1,\"method\":\"eth_getBlockByNumber\",\"params\":[\"$HEX_BLOCK\",false]}" \
    "$RPC_URL" | head -c 200
  echo

  # Test 2: Can the provider serve a balance at that block?
  echo "=== Test 2: Balance at block ==="
  curl -s --max-time 10 -X POST -H "Content-Type: application/json" \
    -d "{\"jsonrpc\":\"2.0\",\"id\":1,\"method\":\"eth_getBalance\",\"params\":[\"0x0000000000000000000000000000000000000000\",\"$HEX_BLOCK\"]}" \
    "$RPC_URL" | head -c 200
  echo

  # Test 3: Can eth_getCode at the fork block?
  echo "=== Test 3: Code at block ==="
  curl -s --max-time 10 -X POST -H "Content-Type: application/json" \
    -d "{\"jsonrpc\":\"2.0\",\"id\":1,\"method\":\"eth_getCode\",\"params\":[\"0x838bf9E95CB12Dd76a54C9f9D2E3082EAF928270\",\"$HEX_BLOCK\"]}" \
    "$RPC_URL" | head -c 200
  echo
'

    If Test 1 passes but Test 2 or 3 fail, the provider doesn't have true archive state — it can serve block headers but not historical state queries. Switch to a paid archive provider.

    Attack Ethereum Block Polygon Block BSC Block
    Nomad Bridge 15,259,100 — —
    Wormhole Guardian 14,125,553 — —
    Poly Network 12,996,693 16,332,982 9,755,172
    Horizon Multisig 14,399,491 — —
    BNB Token Hub — — 22,166,659
    Multichain Router 16,329,241 — —

    Archive availability at these blocks:

    Provider Ethereum (12-16M) Polygon (16M) BSC (9-22M)
    Ankr (free) ✅ Full archive ⚠️ May fail on non-existent addresses ⚠️ May fail on non-existent addresses
    GetBlock (paid $39/mo) ✅ Full archive ✅ Full archive ✅ Full archive
    LlamaNodes (free) ✅ Full archive ❌ No archive at 16M ❌ No archive
    PublicNode (free) ❌ No archive ❌ No archive at 16M ❌ No archive at 9M
    QuickNode (paid) ✅ Full archive ✅ Full archive ✅ Full archive

    Key finding: Even "archive" providers like Ankr may have incomplete historical state for Polygon/BSC when querying non-existent addresses at old blocks. The state root for the fork block is stored, but the full state trie for addresses that didn't exist at that block is not available. This causes anvil_setBalance to fail when creating new addresses.

    Recommended provider setup for multi-chain:

    # testbed.yaml
rpc_providers:
  ethereum:
    - name: ankr_free
      url: "${ETH_RPC_URL}"  # Ankr free tier — reliable for Ethereum
  polygon:
    - name: getblock
      url: "${POLYGON_RPC_URL}"  # GetBlock paid — reliable archive
    - name: ankr
      url: "https://rpc.ankr.com/polygon"
      priority: 2  # fallback
  bsc:
    - name: getblock
      url: "${BSC_RPC_URL}"  # GetBlock paid — reliable archive
    - name: ankr
      url: "https://rpc.ankr.com/bsc"
      priority: 2  # fallback

    "Connection refused" on Fork¶

    ERROR: chain bsc did not become ready within 30s on http://127.0.0.1:8547

    Cause 1: The ${BSC_RPC_URL} env var is empty or points to an unreachable endpoint.

    Fix: Verify the URL in .env and test with curl.

    Cause 2: The fork block is too old for the configured provider.

    Fix: Check the provider's archive depth. For BSC block 9,755,172 (2021), you need a paid archive provider.

    Rate Limited¶

    error code: -32000: rate limit exceeded

    Cause: Free tier rate limits (typically 10-25 req/s).

    Fix: Add more providers as fallbacks in testbed.yaml under rpc_providers.bsc. Crucible will automatically retry with the next provider.

    Summary: What We Need vs What's Free¶

    Chain Block Needed Free Available? Paid Option
    Ethereum 12,996,693 - 15,259,100 [ok] ANKR free tier (sign up) GetBlock/QuickNode
    BSC 9,755,172 [fail] No free provider GetBlock $39/mo
    Polygon 16,332,982 [fail] No free provider GetBlock $39/mo

    Minimum viable paid plan: GetBlock Starter ($39/month) gives access to all 3 chains with archive depth.
