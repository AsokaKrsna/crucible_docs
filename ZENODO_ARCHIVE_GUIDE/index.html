    Zenodo Archive Guide¶

    This guide walks through archiving Crucible on Zenodo for USENIX artifact evaluation.

    Prerequisites¶

    • A GitHub account with push access to the repository
    • A Zenodo account (sign in via GitHub)
    • ORCID (optional but recommended)

    Step 1: Update CITATION.cff¶

    The file CITATION.cff at the project root is already filled with project metadata. Before tagging, fill in your personal information:

    authors:
  - name: Durjoy Majumdar
    email: "your.email@example.com"     # Fill your real email
    orcid: "0000-0000-0000-0000"        # Fill your real ORCID

    Step 2: Link GitHub to Zenodo¶

    1. Go to zenodo.org
    2. Sign in with your GitHub account
    3. Navigate to GitHub in the top menu
    4. Toggle ON for the durjoy-m/crucible-testbed repository
    5. Zenodo will now auto-archive every new release tag

    Step 3: Commit and Tag¶

    # Ensure working tree is clean
git status

# Commit any pending changes
git add -A
git commit -m "v0.4.0 release"

# Tag the release
git tag -a v0.4.0 -m "Crucible v0.4.0 - Blockchain Bridge Security Research Testbed"

# Push the tag
git push origin v0.4.0

    Step 4: Review and Publish on Zenodo¶

    1. Zenodo automatically creates a draft record when it detects the tag push
    2. Wait 5-10 minutes
    3. Visit zenodo.org/deposit
    4. Open the draft record and review the metadata:
    5. Title, authors, description are auto-filled from CITATION.cff
    6. License should be MIT
    7. Version should be 0.4.0
    8. For USENIX AE: Set access to "Restricted" so only evaluators with the private link can access it before publication
    9. Click "Publish" to finalize the record
    10. Zenodo assigns a DOI (e.g., 10.5281/zenodo.XXXXXXX)

    Step 5: Update CITATION.cff with DOI (Post-Tag)¶

    The CITATION.cff at the tagged commit (v0.4.0) won't have the DOI yet. This is normal. After Zenodo assigns the DOI in Step 4:

    1. Update CITATION.cff at the project root:
    # Add at the top level:
doi: 10.5281/zenodo.XXXXXXX
    1. Commit this change (this is a post-tag commit - the tag itself already exists):
    git add CITATION.cff
git commit -m "Add Zenodo DOI to CITATION.cff"
git push

    Step 6: Push and Verify¶

    Push the tag to GitHub:

    git push origin v0.4.0

    Zenodo will automatically create a draft record from the tag. No CI pipeline or GitHub Actions workflow runs — this is a local project.

    What Gets Archived¶

    Zenodo archives the full git repository at the tagged commit:

    Category Contents
    Code 34 Python source files (CLI, plugin system, scenario engine, SDK)
    Tests 249 unit tests, 7 integration tests, CliRunner end-to-end tests
    Docs 27 markdown docs covering architecture, workflow, CLI reference, guides
    Attack Packs 5 packs (Nomad, Poly Network, BNB Token Hub, Horizon, Multichain)
    Docker Docker compose (6 services), Dockerfile
    Evidence Attack pack evidence files with postmortem URLs and fork blocks
    Config pyproject.toml, foundry.toml, testbed.yaml, CITATION.cff
    Experiments 6 experiment configs (demo + 5 attack pack replays)

    Post-Archive Checklist¶

    • CITATION.cff has real email and ORCID
    • GitHub repo is linked to Zenodo (Settings > Repositories)
    • Tag v0.4.0 pushed to GitHub
    • Zenodo draft reviewed and published
    • DOI assigned (check your Zenodo deposit page)
    • CITATION.cff updated with DOI in post-tag commit
    • Tag pushed to GitHub
    • docs/AE_WALKTHROUGH.md DOI link updated
