    Researcher's Guide to Observability & Day-to-Day Commands¶

    Purpose: What a researcher can observe during Crucible experiments and the everyday cast/forge commands they'll need. Covers: Crucible observability commands, JSONL artifact streams, cast usage with forked chains, and troubleshooting.

    Part 1: Observability — What You Can See¶

    Crucible provides multiple windows into what's happening, from real-time CLI status to detailed per-run artifact streams.

    1.1 Quick Health Check: crucible doctor¶

    Run this first to verify your environment is ready:

    crucible doctor
# Sample output:
# Checks: Python imports, Foundry binaries, forge --version,
#         testbed.yaml load, RPC URL resolution, .env file,
#         required directories, port availability, Docker
# Auto-fix: --fix repairs broken symlinks + missing dirs

    1.2 View Running Chains: crucible status¶

    After forking, see every Anvil instance:

    crucible fork --topology single
crucible status

# ┌─────────────────────────────────────────────────┐
# │                    Chains                        │
# ├────────┬────────┬──────┬────────┬────────┬───────┤
# │ Name   │ PID    │ Port │ Block  │ Status  │ RPC  │
# ├────────┼────────┼──────┼────────┼────────┼───────┤
# │ethereum │ 12345  │ 8545 │ 12996693│ ALIVE  │...   │
# └────────┴────────┴──────┴────────┴────────┴───────┘

    Use --disk to see disk usage across states, logs, datasets, and fork cache.

    1.3 What a Run Directory Contains¶

    Every experiment produces a logs/<experiment>/<run_id>/ directory with this structure:

    logs/nomad_replay/DS-2026-00209/
├── run_manifest.json          ← Provenance: git commit, Foundry version,
│                                config fingerprint, cache state, outcome
├── run.jsonl                  ← Experiment lifecycle: phases, steps, messages, errors
├── log_integrity.json         ← SHA-256 hashes of ALL artifact files
├── report.md                  ← Human-readable summary with tables
├── chain_ethereum/            ← Per-chain artifacts (one dir per chain)
│   ├── events.jsonl           ←   Contract events (Transfer, Approval, etc.)
│   ├── gas_profile.jsonl      ←   Gas used per transaction
│   ├── storage_diffs.jsonl    ←   Storage slot changes (before → after)
│   ├── balance_diffs.jsonl    ←   Balance changes per account
│   └── run.jsonl              ←   Chain-specific lifecycle entries
├── bridge/                    ← Cross-chain artifacts (multi-chain only)
│   ├── cross_chain.jsonl      ←   Correlated messages across chains
│   └── events.jsonl           ←   Bridge-specific events
└── artifact_bundle_manifest.json  ← (after export) All files with hashes

    1.4 Inspecting the 6 Artifact Streams¶

    Each stream is JSONL (one JSON object per line):

    run.jsonl — The main event log: 

    {"timestamp":"2026-06-07T...","phase":"setup","action":"experiment_start","status":"running","message":"experiment started"}
{"timestamp":"2026-06-07T...","phase":"attack","action":"send_tx","status":"success","message":"exploit transaction sent"}
{"timestamp":"2026-06-07T...","phase":"teardown","action":"experiment_end","status":"success","message":"experiment completed"}

    events.jsonl — Every emitted contract event: 

    {"chain":"ethereum","event_name":"Transfer","address":"0x...","topics":["0xddf252ad..."],"data":"0x..."}
{"chain":"ethereum","event_name":"MessageAccepted","address":"0x...","topics":["0x8b7c1a1f..."],"data":"0x..."}

    storage_diffs.jsonl — Every slot that changed: 

    {"chain":"ethereum","contract":"0x...","slot":"0x1","before":"0x00","after":"0x01"}

    gas_profile.jsonl — Gas per operation: 

    {"chain":"ethereum","step":"auto_tx","gas_used":25150,"tx_hash":"0x..."}

    balance_diffs.jsonl — Balance changes: 

    {"chain":"ethereum","contract":"0x...","account":"0x...","before":"0x...","after":"0x..."}

    cross_chain.jsonl — Cross-chain message correlation (multi-chain only): 

    {"correlation_id":"xchain-0001","source_chain":"ethereum","dest_chain":"bsc","status":"emitted"}

    1.5 The Report: report.md¶

    After every run, Crucible generates a human-readable report:

    cat logs/nomad_replay/DS-2026-00209/report.md
# Shows:
# - Summary table (log entries, events, storage diffs, gas, failures)
# - Phase breakdown (setup, attack, teardown)
# - Gas profile per step
# - Storage modifications per chain
# - Contract events with counts

    1.6 Integrity Check: log_integrity.json¶

    Every file in the run directory is hashed:

    {
  "run_id": "DS-2026-00209",
  "files": {
    "run.jsonl": "sha256:a1b2c3...",
    "chain_ethereum/events.jsonl": "sha256:d4e5f6...",
    "chain_ethereum/gas_profile.jsonl": "sha256:g7h8i9..."
  }
}

    Use this to verify artifacts haven't been tampered with after export.

    1.7 Comparing Two Runs: crucible diff¶

    The definitive determinism check — proves two runs are identical:

    crucible diff DS-2026-00182 DS-2026-00183
# Output:
# {
#   "storage_equal": true,
#   "events_equal": true,
#   "gas_equal": true,
#   "balances_equal": true,
#   "gas": {"run_a": 25150, "run_b": 25150, "delta": 0},
#   "storage": {"run_a": 2, "run_b": 2},
#   "events": {"run_a": 3, "run_b": 3},
#   "balances": {"run_a": 2, "run_b": 2}
# }

    All four fields must be true for a deterministic reproduction.

    1.8 Browsing History: crucible history¶

    crucible history
# Shows all past runs in a table with:
# Run ID, Experiment, Attack, Outcome, Started At

# Filtering:
crucible history --outcome success
crucible history --tag "nomad"
crucible history --limit 5

# Rebuild index from logs:
crucible history --rebuild

    1.9 The Provenance Record: run_manifest.json¶

    Every run records its exact environment:

    cat logs/nomad_replay/DS-2026-00209/run_manifest.json | jq .provenance
# {
#   "git_commit": "abc123...",
#   "git_dirty": false,
#   "foundry_version": "forge 1.0.0 (stable)",
#   "python_version": "3.12",
#   "docker_mode": false,
#   "config_fingerprint": "sha256:...",
#   "cache_mode": "cold",
#   "experiment_file_hash": "sha256:...",
#   "topology_hash": "sha256:...",
#   "evidence_hash": "sha256:..."
# }

    1.10 The Instrumentation Proxy (Automatic)¶

    When an experiment runs, Crucible places a proxy between the attack plugin and Anvil:

    Attack Plugin → HTTP POST → Instrumentation Proxy (thread) → Forward to Anvil
                                    ↓
                          Auto-captures: gas, events, storage diffs,
                          balance changes — for EVERY transaction

    You don't need to do anything. Every eth_sendTransaction or cast send that flows through the proxy generates complete artifact streams automatically. No per-attack instrumentation code needed.

    1.11 When Artifact Streams Are Empty — Diagnosing the Proxy¶

    If an experiment succeeds (exit 0) but events.jsonl or storage_diffs.jsonl are empty, the proxy's trace capture may have failed silently. This happens when debug_traceTransaction is unavailable or returns an error on the forked Anvil.

    What's affected vs what still works:

    Artifact Captured? Source
    Gas used [ok] Always eth_getTransactionReceipt
    Balance diffs [ok] Usually Captured from trace if available
    Events (logs) [fail] May be empty Captured from receipt if trace available
    Storage diffs [fail] May be empty Requires debug_traceTransaction

    How to recover the data manually using cast:

    # Set your RPC URL
export RPC="http://127.0.0.1:8545"

# 1. Find the transaction hash from the gas profile
cat logs/<experiment>/<run_id>/chain_ethereum/gas_profile.jsonl
# → {"tx_hash": "0x...", "gas_used": 25150}

# 2. Get the receipt — this contains all emitted events (logs)
cast receipt <tx_hash> --rpc-url $RPC
# Look for the "logs" array — each log is an emitted event

# 3. Read the contract's storage to verify state changes
cast storage <contract_address> <slot_hex> --rpc-url $RPC

# 4. Check if debug_traceTransaction is available
cast rpc debug_traceTransaction <tx_hash> \
  '{"tracer": "prestateTracer", "tracerConfig": {"diffMode": true}}' \
  --rpc-url $RPC

    When to worry vs not: - Experiment passes assertions (check run.jsonl) AND gas is captured → Exploit worked, artifacts are bonus - Experiment fails AND artifacts are empty → Something broke before the proxy — check the run log first - Only storage diffs missing but events present → Partial trace capture, still useful

    Root cause check: If debug_traceTransaction works directly against Anvil but not through the proxy, file a bug. If it also fails directly, it's an Anvil version limitation.

    Part 2: Day-to-Day Foundry Commands¶

    Part 2: Day-to-Day Foundry Commands¶

    These are the cast and forge commands you'll use most often while working with Crucible's forked chains.

    2.1 Finding Your Chain's RPC URL¶

    After crucible fork, the default chain runs on:

    # Ethereum (single topology): http://127.0.0.1:8545
# Check with:
crucible status

    Set this as an environment variable for convenience:

    export RPC="http://127.0.0.1:8545"

    2.2 cast Commands — Query Chain State¶

    Check if a contract exists (has code): 

    cast code 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B --rpc-url $RPC
# Returns bytecode if contract exists, "0x" if EOA

    Check an account's ETH balance: 

    cast balance 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B --rpc-url $RPC

    Read storage at a specific slot: 

    cast storage 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B 0x0 --rpc-url $RPC

    Call a read-only contract function: 

    cast call 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B \
  "totalSupply()(uint256)" --rpc-url $RPC

cast call 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B \
  "balanceOf(address)(uint256)" 0x... --rpc-url $RPC

    Get the current block number: 

    cast block-number --rpc-url $RPC

    Get transaction details: 

    cast tx 0x5ea16a4f9104b901a75662f3900ca8d3d92209d846310237ca65675e2f2e519c \
  --rpc-url $RPC

    Get transaction receipt: 

    cast receipt 0x5ea16a4f9104b901a75662f3900ca8d3d92209d846310237ca65675e2f2e519c \
  --rpc-url $RPC

    Decode calldata (with ABI): 

    cast calldata-decode \
  "process(bytes,bytes)" 0x61bd0e46... --rpc-url $RPC

    Get event logs: 

    cast logs --rpc-url $RPC \
  --address 0x98f3c9e6E3fAce36bAAd05FE09d375Ef1464288B \
  --from-block 1 --to-block latest

    Get the 4-byte selector for a function signature: 

    cast sig "transfer(address,uint256)"
# 0xa9059cbb

    Compute keccak256 of arbitrary data: 

    cast keccak "process(bytes,bytes)"
# 0x61bd0e46...

    2.3 cast Commands — Send Transactions¶

    Send ETH: 

    cast send --from 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266 \
  --to 0x... --value 1ether --rpc-url $RPC

    Call a contract function: 

    cast send --from 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266 \
  --to 0x... "transfer(address,uint256)" 0x... 1000 --rpc-url $RPC

    Send raw calldata: 

    cast send --from 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266 \
  --to 0x... --data 0x61bd0e46... --rpc-url $RPC

    2.4 cast Commands — Impersonation¶

    Anvil's --auto-impersonate is enabled by default, so you can impersonate any account:

    # Check cast supports impersonate:
cast rpc anvil_impersonateAccount 0x... --rpc-url $RPC

# Send as the impersonated account:
cast send --from 0x... --unlocked \
  --to 0x... "transfer(address,uint256)" 0x... 1000 --rpc-url $RPC

# Stop impersonating:
cast rpc anvil_stopImpersonatingAccount 0x... --rpc-url $RPC

    2.5 cast Commands — EVM Manipulation¶

    Advance time (for time-dependent exploits): 

    cast rpc evm_increaseTime 86400 --rpc-url $RPC  # +1 day
cast rpc evm_mine --rpc-url $RPC                 # mine a block

    Set storage directly: 

    cast rpc anvil_setStorageAt 0x... 0x0 0x... --rpc-url $RPC

    Set account balance: 

    cast rpc anvil_setBalance 0x... 0x... --rpc-url $RPC

    Dump/load full state: 

    cast rpc anvil_dumpState --rpc-url $RPC > state.json
cast rpc anvil_loadState "$(cat state.json)" --rpc-url $RPC

    2.6 forge Commands — Running Scripts¶

    Run a Solidity script (simulation mode, no broadcast): 

    forge script Exploit.s.sol:Exploit --rpc-url $RPC

    Run and broadcast transactions: 

    forge script Exploit.s.sol:Exploit --rpc-url $RPC --broadcast

    Compile without running: 

    forge build

    2.7 Using the Hybrid Pattern (Solidity + Python)¶

    When forge script --broadcast doesn't work (Foundry simulation differs from Anvil), use this pattern:

    // Exploit.s.sol — Pure calldata computation, no broadcast
contract MyScript is Script {
    function run() external {
        bytes memory data = abi.encodeWithSelector(
            bytes4(0x61bd0e46), arg1, arg2
        );
        console.log("DATA:%s", vm.toString(data));
    }
}

    # Run forge script, capture calldata from console.log output:
forge script Exploit.s.sol:MyScript --rpc-url $RPC 2>&1 | grep "DATA:"

# Then send the calldata via cast:
cast send --from 0x... --to 0x... --data <calldata> --rpc-url $RPC

    2.8 Useful Anvil Flags (for manual runs)¶

    When running Anvil directly outside Crucible:

    anvil \
  --fork-url $ETH_RPC_URL \
  --fork-block-number 12996693 \
  --host 127.0.0.1 \
  --port 8545 \
  --chain-id 1 \
  --gas-price 0 \
  --block-base-fee-per-gas 1000000000 \
  --auto-impersonate \
  --order fifo \
  --timestamp "fork_block"    # "fork_block" for deterministic; $(date +%s) for live

    Part 3: Typical Researcher Workflow¶

    Morning: Set Up¶

    # 1. Check environment
crucible doctor

# 2. Clean previous state
crucible clean --states

# 3. Fork the chain
crucible fork --topology bridge
crucible status  # verify chains are alive

# 4. Run your experiment
crucible experiment run experiments/nomad_replay.yaml --no-baseline

    During: Observe¶

    # Watch the run manifest for real-time provenance
tail -f logs/nomad_replay/DS-2026-00209/run.jsonl

# Or inspect artifacts after
cat logs/nomad_replay/DS-2026-00209/report.md

# Query chain state directly
cast call 0x... "committedRoot()(bytes32)" --rpc-url $RPC
cast balance 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266 --rpc-url $RPC

    Afternoon: Verify + Diff¶

    # Run a second time
crucible experiment run experiments/nomad_replay.yaml --no-baseline

# Verify determinism
crucible diff DS-2026-00182 DS-2026-00183
# Expected: all 4 fields equal = true

# Browse history
crucible history

# Export for publication
crucible export DS-2026-00182
crucible verify datasets/DS-2026-00182/

    Quick Reference Card¶

    Crucible Commands¶

    Command What It Does When to Use
    crucible doctor Validate environment First thing daily
    crucible status Show running Anvil processes After fork
    crucible history Browse past runs Track progress
    crucible diff <a> <b> Compare two runs Verify determinism
    crucible clean Remove artifacts Before fresh start
    crucible export <id> Create publication bundle When publishing

    Cast Commands¶

    Command Purpose
    cast balance <addr> Check ETH balance
    cast code <addr> Check if contract exists
    cast storage <addr> <slot> Read storage at slot
    cast call <addr> "sig" <args> Call read-only function
    cast send <addr> "sig" <args> Send transaction
    cast tx <hash> Get transaction details
    cast receipt <hash> Get transaction receipt
    cast logs --address <addr> Filter event logs
    cast block-number Current block
    cast sig "func(args)" Get 4-byte selector
    cast keccak "data" Compute keccak256
    cast rpc <method> <params> Call any RPC method

    Anvil RPC Methods (via cast rpc)¶

    Method Purpose
    anvil_impersonateAccount Impersonate any address
    anvil_stopImpersonatingAccount Stop impersonation
    anvil_setBalance Change account balance
    anvil_setStorageAt Write storage directly
    anvil_dumpState Snapshot full chain state
    anvil_loadState Restore from snapshot
    evm_increaseTime Advance timestamp
    evm_mine Mine a new block
